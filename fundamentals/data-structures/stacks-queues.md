# Stacks and Queues - 栈与队列

> 限制访问方式的线性结构：后进先出 (LIFO) vs 先进先出 (FIFO)

## 🎯 核心区别

| 特性 | 栈 (Stack) | 队列 (Queue) |
| :--- | :--- | :--- |
| **原则** | 后进先出 (LIFO) | 先进先出 (FIFO) |
| **插入/删除** | Push / Pop | Enqueue / Dequeue |
| **典型应用** | 撤销、函数调用栈、括号匹配 | 任务调度、BFS、缓冲处理 |

---

## 📚 栈 (Stack)
**定义**：只允许在末尾（栈顶）进行插入和删除操作的线性结构。

*   **核心特性**：
    *   **LIFO**：最后进入的元素最先被处理。
    *   **复杂度**：Push 和 Pop 均为 **O(1)**。
*   **实现建议**：Python 中直接使用 `list` 即可（末尾操作为 $O(1)$ 摊销）。
*   **关键场景**：
    *   **函数调用栈**：管理递归过程中的返回地址与局部变量。
    *   **撤销操作 (Undo)**：记录历史操作序列。
    *   **括号匹配**：通过压入左括号、弹出匹配右括号实现。

## 🚶 队列 (Queue)
**定义**：只允许在尾部插入，头部删除的线性结构。

*   **核心特性**：
    *   **FIFO**：最早进入的元素最先被处理。
    *   **复杂度**：Enqueue 为 **O(1)**，Dequeue 需选对底层实现。
*   **实现建议**：**务必使用 `collections.deque`**。使用 `list.pop(0)` 是 $O(n)$ 的，效率极低。
*   **关键场景**：
    *   **任务调度**：异步任务队列、打印作业、客服请求。
    *   **广度优先搜索 (BFS)**：按层遍历图或树。

## 🔄 变种数据结构
*   **双端队列 (Deque)**：支持在两端高效插入和删除，常用于滑动窗口类问题。
*   **优先队列 (Priority Queue)**：不再遵循 FIFO，而是按“优先级”出队（通常基于堆实现，复杂度 **O(log n)**）。

---

## 📊 性能速查表

| 数据结构 | 插入 (尾部) | 删除 (头部/栈顶) | 访问 | 空间 |
| :--- | :--- | :--- | :--- | :--- |
| **栈 (list)** | O(1) | O(1) | O(1) | O(n) |
| **队列 (deque)** | O(1) | O(1) | O(n) | O(n) |
| **优先队列 (heap)** | O(log n) | O(log n) | O(1) | O(n) |
