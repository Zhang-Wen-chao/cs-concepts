# Trees and Graphs - 树与图

> 层级关系和网络关系的数据结构。

## 🌳 树 (Tree)
**定义**：具有层级关系的非线性结构。
*   **根 (Root)**：顶端节点。
*   **子/叶子节点**：拥有分支的节点 vs 无子节点的节点。
*   **遍历逻辑**：
    *   **前/中/后序**：DFS（深度优先）的递归变体。
    *   **层序**：BFS（广度优先）。

### 特殊树结构
*   **二叉搜索树 (BST)**：左子树 < 根 < 右子树。搜索效率 $O(\log n)$（需平衡）。
*   **平衡树 (AVL/红黑树)**：通过旋转维持高度平衡，保证最坏情况 $O(\log n)$。
*   **堆 (Heap)**：完全二叉树，用于优先队列。堆顶始终为最大/最小值。
*   **字典树 (Trie)**：前缀树，用于高效存储和查找字符串前缀（路径共享）。

## 🕸️ 图 (Graph)
**定义**：由一组顶点 (Vertex) 和连接顶点的边 (Edge) 组成的结构。

### 核心表示法
*   **邻接矩阵**：二维数组，$O(V^2)$ 空间。适合稠密图，边查询快 $O(1)$。
*   **邻接表**：哈希表/链表组合，$O(V+E)$ 空间。适合稀疏图，遍历快。

### 核心算法本质
*   **DFS (深度优先)**：使用栈（递归），适合拓扑排序、连通性检查。
*   **BFS (广度优先)**：使用队列，适合最短路径（无权图）。
*   **Dijkstra**：贪心策略，结合优先队列实现加权最短路径。

## 📊 性能速查

| 结构 | 查找/搜索 | 插入 | 删除 |
| :--- | :--- | :--- | :--- |
| **BST (平衡)** | $O(\log n)$ | $O(\log n)$ | $O(\log n)$ |
| **堆 (Heap)** | $O(n)$ | $O(\log n)$ | $O(\log n)$ |
| **图 (遍历)** | $O(V+E)$ | - | - |
