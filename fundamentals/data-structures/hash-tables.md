# Hash Tables - 哈希表

> 快速查找的利器：通过哈希函数将键映射到数组索引，实现 O(1) 的平均查找时间。

## 🎯 核心原理
哈希表的核心是**哈希函数 (Hash Function)**。它将任意大小的键（Key）映射为固定范围的数组索引（Index）：
`Key → Hash Function → Index → Value`

*   **理想情况**：每个键对应唯一索引，查找/插入/删除均需 **O(1)**。
*   **实际挑战**：不同的键可能映射到相同的索引，这就是**哈希冲突 (Collision)**。

---

## ⚔️ 冲突解决方法

### 1. 链地址法 (Chaining)
在发生冲突的数组槽位中存储一个链表（或数组），所有映射到该索引的元素都串联在链表中。
*   **优点**：实现简单，删除方便。
*   **缺点**：额外的指针内存开销。

### 2. 开放寻址法 (Open Addressing)
不使用额外结构，如果发生冲突，则在哈希表中寻找下一个空的槽位。
*   **常用策略**：
    *   **线性探测**：`index + 1, index + 2, ...`
    *   **二次探测**：`index + 1², index + 2², ...`
    *   **双重哈希**：使用第二个哈希函数确定步长。

---

## 📊 负载因子 (Load Factor)

**负载因子** = 已存储元素数量 / 哈希表总容量

*   **机制**：负载因子过高（通常 > 0.75）会导致查找效率急剧下降，此时哈希表会触发**动态扩容**（通常翻倍），并重新哈希所有元素。
*   **权衡**：负载因子越小，冲突越少（快），但空间浪费越多；负载因子越大，空间利用率高，但冲突概率增加（慢）。

---

## 💡 使用场景速查

### ✅ 适合场景
*   **快速查找**：需要从大量数据中定位特定对象。
*   **去重/计数**：判断元素是否存在，或统计词频。
*   **缓存实现**：如 Python 的 `dict`、LRU 缓存。

### ❌ 不适合场景
*   **有序遍历**：哈希表是无序的，如果需要范围查询或有序访问，请使用**树 (Tree)**。
*   **内存敏感**：哈希表通常需要预留空间（保持较低负载因子）。

---

## 📚 Python 的实现视角
Python 的 `dict` 和 `set` 均是高度优化的哈希表。
*   **可哈希对象**：必须是不可变对象（如数字、字符串、元组），因为哈希值必须固定。
*   **性能保证**：Python 自动管理扩容与负载因子，用户无需手动调优。

---

**记住**：
1. 哈希表 = 空间换时间的典范。
2. 平均 O(1) 查找，但最坏情况 O(n)（发生严重冲突时）。
3. 没有完美的哈希函数，好的哈希函数应尽量让数据均匀分布。
