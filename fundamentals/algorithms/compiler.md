# Compiler - 编译器

> 把高级语言翻译成机器指令的程序

## 🎯 核心思想

**编译器**将人类可读的源代码逐层降级，直到变成机器可执行的指令。

```
源代码 (.c/.go/.py)
    │
    ▼
词法分析 (Lexical Analysis)    ─→ 字符流 → Token 序列
    │
    ▼
语法分析 (Syntax Analysis)     ─→ Token 序列 → 抽象语法树 (AST)
    │
    ▼
语义分析 (Semantic Analysis)   ─→ 类型检查、作用域解析
    │
    ▼
中间代码生成 (IR Generation)   ─→ AST → 平台无关 IR（三地址码/SSA）
    │
    ▼
优化 (Optimization)            ─→ 常量折叠、死代码消除、循环优化
    │
    ▼
代码生成 (Code Generation)     ─→ IR → 目标机器指令（x86/ARM/RISC-V）
```

---

## 📚 各阶段详解

### 1. 词法分析 (Lexer)
- **输入**：字符流（源代码文本）
- **输出**：Token 流（关键字、标识符、字面量、运算符）
- **工具**：正则表达式、有限自动机（DFA/NFA）
- **例子**：`int a = 3 + 5;` → `TYPE(int) ID(a) ASSIGN INT(3) PLUS INT(5) SEMI`

### 2. 语法分析 (Parser)
- **输入**：Token 流
- **输出**：抽象语法树（AST）
- **工具**：上下文无关文法（CFG）、LL/LR 解析器
- **核心**：递归下降解析（手写）、Yacc/Bison（生成）

### 3. 语义分析
- **输入**：AST
- **输出**：带类型/作用域信息的 AST（注解树）
- **检查**：类型匹配、变量声明确认、作用域规则
- **例子**：`"hello" + 42` → 类型错误

### 4. 中间代码 (IR)
- **三层 IR 设计**：
  - HIR（高级 IR）— 接近 AST，保留循环/函数等结构
  - MIR（中级 IR）— 显式控制流
  - LIR（低级 IR）— 接近目标机器的寄存器/指令

- **SSA（Static Single Assignment）**：每个变量只赋值一次，简化优化

| IR 形式 | 抽象层级 | 例子 |
|---------|---------|------|
| AST | 最高 | `BinOp(Add, Int(3), Int(5))` |
| 三地址码 | 中间 | `t1 = 3 + 5` |
| SSA | 中间 | `t1_1 = add 3, 5` |
| 汇编 | 最低 | `add eax, 3` |

### 5. 优化 (Optimization)

| 优化 | 做法 | 效果 |
|------|------|------|
| 常量折叠 | `3 + 5` → `8` | 消除编译期已知的计算 |
| 死代码消除 | 删除无用的赋值 | 减小体积 |
| 公共子表达式 | `a + b` 算一次复用 | 减少重复计算 |
| 循环不变式外提 | 循环内不变的计算移出去 | 加速循环 |
| 内联展开 | 函数调用替换为函数体 | 减少调用开销 |
| 寄存器分配 | 变量映射到物理寄存器 | 减少内存访问 |

### 6. 代码生成
- **指令选择**：IR → 目标机器指令
- **寄存器分配**：图着色算法
- **指令调度**：重排指令充分利用流水线

## 🔗 概念关系图

```
自动机/形式语言 (理论)
    │
    ├── 正则语言 ←→ 词法分析 (Lexer)
    ├── 上下文无关文法 ←→ 语法分析 (Parser)
    └── 下推自动机 ←→ 语法树构建

优化 ←→ 循环变换、数据流分析
寄存器分配 ←→ 图着色算法（NP 难）
指令调度 ←→ 流水线冒险（计算机体系结构）
```

---

## 💡 关键洞察

1. **编译器就是翻译器** — 不是"理解"代码，而是机械地逐层降级
2. **前端的语言相关，后端的目标相关** — LLVM 的架构就是复用前端和后端，中间是统一 IR
3. **优化是核心价值** — 同一个 C 代码，gcc -O0 vs -O3 性能差几倍
4. **SSA 是现代编译器的基础** — LLVM、GCC 都基于 SSA IR，让优化更简单
