# Dynamic Programming - 动态规划

> 记住过去，避免重复计算。

## 🎯 核心概念
**动态规划 (DP)** 通过将原问题分解为简单的子问题，并存储其结果（记忆化），从而避免重复计算。

*   **三个要素**：
    *   **重叠子问题**：相同问题被反复求解。
    *   **最优子结构**：全局最优解包含子问题的最优解。
    *   **状态转移方程**：描述子问题间递推关系的数学表达式。

---

## 🛠️ 解题五步法
1.  **定义状态**：`dp[i]` 或 `dp[i][j]` 代表什么含义？
2.  **状态转移方程**：`dp[i]` 如何从 `dp[i-1], dp[i-2]...` 推导而来？
3.  **初始条件**：最简单的边界情况（如 `dp[0]` 或 `dp[0][0]`）。
4.  **计算顺序**：通常从小到大 (Bottom-Up) 或递归 (Top-Down)。
5.  **结果返回**：最终目标 `dp[n]` 或 `dp[m][n]`。

---

## 📈 经典模型 (状态转移方程)

| 问题 | 状态转移方程 (核心本质) |
| :--- | :--- |
| **爬楼梯** | `dp[i] = dp[i-1] + dp[i-2]` |
| **打家劫舍** | `dp[i] = max(dp[i-1], dp[i-2] + nums[i])` |
| **0-1 背包** | `dp[w] = max(dp[w], dp[w-weight[i]] + value[i])` |
| **最长递增子序列** | `dp[i] = max(dp[j]) + 1` (其中 `nums[j] < nums[i]`) |
| **最长公共子序列** | 若 `s1[i]==s2[j]` 则 `dp[i-1][j-1]+1`；否则 `max(dp[i-1][j], dp[i][j-1])` |
| **编辑距离** | 若不等：`1 + min(插入, 删除, 替换)` |

---

## 💡 优化技巧
*   **空间压缩**：如果状态只依赖于前一行或前两个值，可使用一维数组甚至常数变量替代二维数组（滚动数组优化）。
*   **记忆化递归 (Memoization)**：自顶向下解决，用哈希表/数组缓存结果。
*   **状态压缩**：当状态为布尔值或集合时，可用位运算进行压缩。

---

## 🎯 如何识别 DP 问题？
*   **信号**：求“最值”（最大/最小）、求“方案数”、“是否存在”。
*   **结构**：明确的子问题分解，且子问题间存在明显的依赖关系。
*   **决策**：每一层都有一个选择（选或不选、放或不放）。

**记住**：DP 的本质是**“用空间换时间”**，核心在于找到那个最优的**状态转移方程**。
