# Dynamic Programming - 动态规划

> 记住过去，避免重复计算。

## 🎯 核心概念

**动态规划 (DP)** 通过分解问题并存储子问题结果（记忆化），避免计算冗余。

* **三个要素**：
 * **重叠子问题**：相同子问题被反复求解。
 * **最优子结构**：全局最优解包含子问题的最优解。
 * **状态转移方程**：描述子问题间递推关系的数学表达式。

---

## 🚦 如何识别 DP

| 信号 | 含义 |
|------|------|
| 求**最值**（最大/最小） | 大概率 DP |
| 求**方案数** | 大概率 DP |
| 问**是否存在** | 可能是 DP |
| 每一步有**选择**（选/不选） | DP vs 贪心 |
| 暴力递归有**大量重复子问题** | 必是 DP |

**关键判别**：暴力递归 → 画递归树 → 有重叠子问题 → DP

---

## 🔗 概念关系图

```
暴力递归 (Brute Force)
   │
   ├── 有重叠子问题 ──→ 记忆化递归 (Memoization) ──→ 动态规划 (DP)
   │                          │                              │
   │                      Top-Down                     Bottom-Up
   │                      递归+缓存                     迭代+表格
   │
   └── 无重叠子问题 ──→ 分治 (Divide & Conquer)
                          │
                     归并/快速排序、树的遍历


DP 与 贪心 (Greedy) 的边界：

    都有最优子结构
    DP      ←→      贪心
  选所有可能           选眼前最优
  保证全局最优         局部最优≠全局最优
```

---

## 💡 核心思想

* **空间换时间**：用数组/哈希表存子问题结果，避免重复计算
* **Top-Down (记忆化递归)**：直觉驱动，递归定义 → 加缓存
* **Bottom-Up (迭代填表)**：从小到大的顺序计算，效率更高
* **空间压缩**：若状态只依赖前几行，用滚动数组降维

**本质**：暴力递归 + 缓存 = 记忆化递归 + 系统化填表 = DP
