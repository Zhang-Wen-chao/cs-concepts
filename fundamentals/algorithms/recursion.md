# Recursion - 递归

> 函数调用自己：分解问题，解决子问题，组合答案

## 🎯 核心概念

**递归**是函数直接或间接调用自己的编程技术。

### 三要素

1. **基线条件** — 最简单的情况，直接返回（停止条件）
2. **递归条件** — 将问题分解成更小的子问题
3. **向基线靠近** — 每次递归调用都要更接近基线

---

## 🔄 递归的本质

```
问题 = 子问题 + 合并
       ↓
  再分解，直到基线
```

**执行模型**：调用栈（后进先出）
- 每层调用压栈
- 到达基线后逐层弹出
- 深度过大 → 栈溢出

---

## 🔗 概念关系图

```
递归
  │
  ├── 有重叠子问题 ──→ 记忆化递归 ──→ DP
  │
  ├── 无重叠子问题 ──→ 分治（归并/快排/树的遍历）
  │
  ├── 多分支试错 ──→ 回溯（全排列/N皇后）
  │
  └── 回溯 + 剪枝 ──→ 分支限界
```

---

## 💡 递归 vs 迭代

| | 递归 | 迭代 |
|--|------|------|
| **空间** | O(n) 栈空间 | O(1) |
| **代码** | 吻合问题定义 | 显式控制 |
| **适用** | 树/图/分治/回溯 | 简单循环 |

**选择指南**：
- 问题有天然递归定义 → 递归
- 深度可能很大 → 迭代
- 重复计算多 → 加记忆化或改 DP
