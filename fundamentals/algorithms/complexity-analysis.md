# Complexity Analysis - 复杂度分析

> 衡量算法效率：时间多快？空间多用？

## 📊 大 O 表示法

**描述算法效率随输入规模增长的趋势**（不是精确运行时间）。

| 复杂度 | 名称 | 增长速度 | 可处理规模 |
|-------|------|---------|-----------|
| **O(1)** | 常数 | 不变 | 不限 |
| **O(log n)** | 对数 | 极慢增长 | 极大 |
| **O(n)** | 线性 | 与输入成正比 | ~10⁸ |
| **O(n log n)** | 线性对数 | 略快于线性 | ~10⁶ |
| **O(n²)** | 平方 | 快 | ~10⁴ |
| **O(2ⁿ)** | 指数 | 极快 | ~20 |
| **O(n!)** | 阶乘 | 灾难 | ~11 |

---

## 🧮 核心规则

| 规则 | 举例 | 解释 |
|------|------|------|
| **忽略常数** | O(3n) → O(n) | 常数项不重要 |
| **只看最高阶** | O(n² + n) → O(n²) | 低阶项被淹没 |
| **独立变量分开** | O(m + n) | m 和 n 是不同输入 |
| **最坏情况优先** | 快排 O(n²) | 给性能保证 |

---

## 💾 空间复杂度

| 来源 | 复杂度 | 例子 |
|------|-------|------|
| 固定变量 | O(1) | `total = 0` |
| 额外数组 | O(n) | 复制数组 |
| 递归栈 | O(n) | 深度 n 的递归 |
| 二维矩阵 | O(n²) | n×n 矩阵 |

---

## ⚖️ 时间与空间的权衡

```
内存更多 ↔ 速度更快  (用记忆化缓存换 O(2ⁿ) → O(n))
内存更少 ↔ 速度更慢  (用 O(1) 空间做原地算法)
```

**摊销分析**：单次操作可能 O(n)，但平均是 O(1)（如动态数组扩容）。
