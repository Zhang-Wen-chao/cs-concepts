# Monte Carlo Tree Search - 蒙特卡洛树搜索

> 用随机模拟驱动搜索，在巨大搜索空间中高效找到最优决策

## 🎯 核心思想

MCTS 结合了**树搜索**的精确性和**蒙特卡洛模拟**的随机性，不需要领域知识就能在巨大状态空间中做出决策。

**AlphaGo 击败李世石的核心算法。**

---

## 🔄 四步循环

```
         选择 (Select)              扩展 (Expand)         模拟 (Simulate)        反向传播 (Backprop)
              │                          │                      │                      │
    ┌───── UCB 选最优子节点 ────┐  未探索节点 ──→  随机 playout  ──→  结果回溯到祖先
    │                          │     加入树         直到终局          更新胜率/访问数
    ▼                          ▼                     ▼                      ▼
    [根] ──→ [子] ──→ [孙] ──→ [新] ──→ 随机走子/终局 ──→ 更新路径上所有节点
```

### 1. 选择 (Select)
从根节点出发，用 **UCB (Upper Confidence Bound)** 公式选择子节点，直到叶子节点：

$$UCB = \frac{w_i}{n_i} + c \sqrt{\frac{\ln N}{n_i}}$$

- $w_i/n_i$ = 胜率（利用 Exploitation）
- $\sqrt{\ln N / n_i}$ = 探索度（探索 Exploration）
- $c$ = 探索常数（越大越倾向探索）

### 2. 扩展 (Expand)
如果叶子节点不是终局，创建它的子节点。

### 3. 模拟 (Simulate)
从新子节点开始，随机走子（rollout/playout）直到终局，记录胜负。

### 4. 反向传播 (Backpropagate)
将模拟结果沿路径更新回根节点：访问次数 +1，胜场 + 结果。

## 📊 MCTS vs 其他搜索

| 算法 | 领域知识 | 搜索空间 | 保证 | 典型应用 |
|------|---------|---------|------|---------|
| 穷举搜索 | 不依赖 | 极小 | 最优解 | 井字棋 |
| 极大极小 + α-β | 需要评估函数 | 中等 | 最优解 | 国际象棋 |
| **MCTS** | 不需要 | 巨大 | 渐近最优 | 围棋、Atari |
| 回溯 | 依赖剪枝条件 | 中-大 | 穷举 | 排列/组合 |

## 🔗 概念关系图

```
暴力搜索 (Brute Force)
    │
    ├── 有领域知识 ──→ 极大极小 + α-β 剪枝 (Minimax)
    │
    ├── 无领域知识 ──→ 蒙特卡洛树搜索 (MCTS)
    │                       │
    │                       ├── 蒙特卡洛模拟 (随机采样)
    │                       └── 树搜索 (UCB 选择)
    │
    └── 搜索 + 学习 ──→ AlphaZero (MCTS + 神经网络)
                              │
                        策略网络引导选择
                        价值网络替代模拟
```

---

## 💡 关键洞察

1. **不对称树** — MCTS 会更深地探索有希望的路径，而不是均匀展开（和 Minimax 不同）
2. **渐近收敛** — 模拟次数越多，MCTS 的决策越接近最优
3. **并行友好** — 每次模拟独立，可并行执行
4. **AlphaZero 的演进** — 用神经网络替代随机模拟和价值估计，让 MCTS 更强大
