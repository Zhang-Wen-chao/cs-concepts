# Backtracking - 回溯

> 试错：走不通就退回来换条路再走。

## 🎯 核心概念

**回溯**是"状态重置"版本的暴力枚举。每一步做选择→递归→撤销选择。

### 和普通递归的区别

| | 普通递归 | 回溯 |
|--|---------|------|
| **过程** | 一路到底 | 试所有可能路径 |
| **控制** | 自然归约 | 显式"选择→撤销" |
| **状态** | 不变/只读 | 现场修改→恢复 |
| **空间** | 调用栈 | 调用栈 + 状态副本/恢复 |

### 算法框架（模板）
```python
def backtrack(路径, 选择列表):
    if 满足结束条件:
        记录结果
        return

    for 选择 in 选择列表:
        # 做选择
        做选择
        backtrack(路径, 选择列表)
        # 撤销选择
        撤销选择
```

## 📊 迭代的多种方向

你说的对——迭代不止"从小到大"。

| 迭代方向 | 例子 | 操作 |
|---------|------|------|
| **从小到大** | DP 填表 | `for i in range(1, n)` |
| **从大到小** | 背包一维压缩 | `for w in range(cap, 0, -1)` |
| **缩小范围** | 二分查找 | `lo, hi` 逐步收窄 |
| **分而治之** | 快排分片 | 递归分治，但分隔本身迭代 |
| **双向逼近** | 两数之和双指针 | `i, j` 从两端向中间 |
| **逐层扩散** | BFS | 队列一层一层展开 |
