# Error Handling - 错误处理

> 优雅地处理错误，让程序更健壮。

## 🎯 错误类型

| 类型 | 产生时机 | 可处理性 | 示例 |
|------|----------|----------|------|
| **语法错误** | 编译/解析时 | ❌ 必须修正 | 少括号、`if x = 5` |
| **运行时错误** | 运行时 | ✅ try/catch | 除以零、下标越界 |
| **逻辑错误** | 运行时 | ⚠️ 需调试 | 计算结果不对、算法错误 |

**错误处理主要针对：运行时错误**

---

## 🔧 处理策略对比

| 策略 | 工作方式 | 优点 | 缺点 |
|------|----------|------|------|
| **异常机制** | try → throw → catch → finally | 清晰，可传播 | 控制流隐式 |
| **错误码** | 返回 int/状态码判断 | 简单直接 | 调用方易忽略 |
| **Result 类型** | `Ok(value) \| Err(error)` | 类型系统保障 | 需模式匹配 |
| **Option 类型** | `Some(value) \| None` | 消灭空指针 | 仅二值 |

## 💡 最佳实践

### 原则清单

| 原则 | 说明 |
|------|------|
| **尽早检查，快速失败** | 前置校验所有条件 |
| **具体异常类型** | 不要笼统 `except Exception` |
| **不要吞异常** | 至少日志记录，选择性重抛 |
| **清理资源** | `try/finally` 或 `with` 上下文管理器 |
| **异常转换** | 低层异常 → 业务含义异常 |
| **重试机制** | 网络/IO 异常可自动重试（指数退避） |
| **降级策略** | 主链路失败 → 缓存 → 默认值 |

### 陷阱清单

```
❌ 空值不检查               →  ✅ Option/Optional 类型
❌ 索引不检查               →  ✅ 先判空/判边界
❌ 类型不验证               →  ✅ isinstance 或类型注解
❌ 文件/网络资源不关闭       →  ✅ 上下文管理器 (with)
❌ 吞掉异常/用 pass         →  ✅ 记录日志或重抛
```
