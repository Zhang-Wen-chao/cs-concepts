# Memory Management - 内存管理

> 程序如何使用内存？分配与释放的策略？如何避免泄漏？

## 🎯 核心概念

**内存管理**是程序运行时使用和管理计算机内存的过程：分配 → 使用 → 释放。

---

## 🏗️ 内存结构

```
+-------------------+  高地址
|       栈          |  ← 局部变量、函数调用 (自动管理)
|       ↓           |
|                   |
|       ↑           |
|       堆          |  ← 动态分配 (手动/GC管理)
+-------------------+
|   BSS段(未初始化)  |  ← 未初始化全局变量
+-------------------+
|   数据段(已初始化) |  ← 已初始化全局变量
+-------------------+
|     代码段         |  ← 程序代码
+-------------------+  低地址
```

## 📦 栈 vs 堆

| 特性 | 栈 (Stack) | 堆 (Heap) |
|------|-----------|-----------|
| **管理方式** | 自动（编译期） | 手动/GC |
| **速度** | 快（移动指针） | 慢（查找空闲块） |
| **容量** | 有限（几 MB） | 大（几 GB） |
| **生命周期** | 函数作用域内 | 程序员控制 |
| **数据结构** | LIFO | 任意 |
| **典型内容** | 局部变量、参数、返回地址 | 对象、大型结构 |

---

## 🔄 分配方式对比

| 方式 | 时机 | 示例 |
|------|------|------|
| **静态分配** | 编译期确定位置和大小 | 全局变量、static |
| **自动分配** (栈) | 进入作用域时 | 局部变量 |
| **动态分配** (堆) | 运行时按需 | `malloc` / `new` / 对象创建 |

## 🗑️ 释放策略

| 策略 | 工作方式 | 代表语言 | 优点 | 缺点 |
|------|----------|----------|------|------|
| **手动管理** | 程序员调用 free/delete | C/C++ | 性能可控 | 易泄漏/悬空指针 |
| **引用计数** | 对象被引用计数归零时释放 | Python, Swift | 实时性 | 循环引用需额外处理 |
| **标记-清除** | 从 GC Root 出发标记可达对象 | Java, Go | 处理循环引用 | Stop-The-World |
| **分代回收** | 新对象频繁检查，老对象少检查 | Java (HotSpot) | 效率高 | 调优复杂 |
| **所有权系统** | 编译时生命周期检查，零开销 | Rust | 安全 + 零开销 | 学习曲线 |

## 🐛 常见问题

| 问题 | 描述 |
|------|------|
| **内存泄漏** | 分配不释放，程序内存持续增长 |
| **悬空指针** | 指向已释放内存，访问导致未定义行为 |
| **缓冲区溢出** | 写入超出分配边界 |
| **双重释放** | 同一块内存释放两次 |
| **栈溢出** | 无限递归 / 局部变量过大 |
