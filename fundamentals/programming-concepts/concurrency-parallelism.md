# Concurrency and Parallelism - 并发与并行

> 同时处理多个任务。并发是结构，并行是执行。

## 🎯 核心概念

### 并发 (Concurrency) — 同时处理多个任务，不一定同时执行
```
厨师一人：煮面→等水开时切菜→炒菜→等锅热时洗碗 → 任务交替
```

### 并行 (Parallelism) — 真正同时执行多个任务
```
多个厨师：A煮面 B切菜 C洗碗 → 任务真正同时运行
```

---

## 🔍 对比

| 特性 | 并发 | 并行 |
|------|------|------|
| **定义** | 处理多个任务 | 执行多个任务 |
| **核心** | CPU 时间片轮转 | 多个 CPU 同时工作 |
| **硬件** | 单核也可以 | 需多核 |
| **目的** | 提高响应性 | 提高吞吐量 |
| **例子** | 浏览器多标签页 | 视频渲染 |

> *"Concurrency is about dealing with lots of things at once. Parallelism is about doing lots of things at once."* — Rob Pike

## 🧵 线程 vs 进程

| 特性 | 进程 | 线程 |
|------|------|------|
| **内存空间** | 独立 | 共享 |
| **创建开销** | 大 | 小 |
| **隔离性** | 强 | 弱 |
| **通信方式** | IPC（管道/套接字） | 共享变量/锁 |
| **安全** | 互不影响 | 易出现竞态 |

## 🔐 同步原语速览

| 原语 | 用途 | 说明 |
|------|------|------|
| **Lock/Mutex** | 互斥访问 | 同时仅一个线程进入 |
| **Semaphore** | 限流 | 指定数量并发访问（如连接池） |
| **Condition** | 线程间信号 | wait/notify 模式 |
| **Event** | 简单标志 | 一个线程触发，多个等待 |

### 常见问题

| 问题 | 描述 |
|------|------|
| **竞态条件** | 多线程同时读写共享数据，结果依赖执行时序 |
| **死锁** | A 等 B 释放锁，B 等 A 释放锁，互相等待 |
| **活锁** | 两个线程不断重试但都过不去 |
| **饥饿** | 某线程一直拿不到资源 |

**死锁四条件**：互斥 · 持有并等待 · 不可抢占 · 循环等待

## 🎯 选择指南

| 场景 | 推荐 |
|------|------|
| **I/O 密集型** (网络/文件/DB) | 异步编程 > 多线程 |
| **CPU 密集型** (计算/渲染) | 多进程 |
| **混合** | 多进程 + 异步事件循环 |

---

## 💡 最佳实践

1. **尽量避免共享状态** — 用消息传递替代共享内存
2. **使用高级抽象** — 线程池/进程池替代手动管理
3. **不可变数据优先** — 无需锁，天然线程安全
4. **正确关闭资源** — executor.shutdown(wait=True)
