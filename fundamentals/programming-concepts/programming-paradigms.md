# Programming Paradigms - 编程范式

> 不同的编程思维方式：如何组织代码、如何思考问题。

## 🎯 核心概念

编程范式是**编程的思维方式**，定义了代码组织、计算表达和问题抽象的方式。**同一个语言可支持多种范式。**

---

## 🔍 五大范式概览

| 范式 | 核心思想 | 关键词 | 适合场景 |
|------|----------|--------|----------|
| **命令式** | 告诉计算机"怎么做" | 语句、状态修改、控制流 | 系统脚本、性能关键 |
| **声明式** | 告诉计算机"要什么" | 结果描述、隐藏过程 | SQL、配置、数据管道 |
| **面向对象 (OOP)** | 用对象建模世界 | 封装、继承、多态、抽象 | 大型业务系统、UI |
| **函数式 (FP)** | 计算即函数求值 | 纯函数、不可变性、高阶函数 | 数据处理、并发 |
| **过程式** | 用过程/函数组织代码 | 自顶向下、数据与算法分离 | 中小型程序 |

## 🔗 概念关系图

```
              编程范式 (思维层次)
                    │
     ┌──────────────┼──────────────┐
     ▼              ▼              ▼
  命令式          声明式          函数式
  ┌─────┐       ┌─────┐        ┌──────┐
  │ 怎么做 │    │ 要什么 │     │ 纯函数  │
  │ 状态变化│    │ 系统负责│    │ 不可变  │
  └──┬──┘       └─────┘        └──────┘
     │
  ┌──┴────┐
  过程式    OOP
  ┌────┐  ┌──────────┐
  │函数│  │对象+类    │
  │无类│  │封装/继承  │
  └────┘  │多态/抽象  │
          └──────────┘
```

## 🧱 OOP 四大支柱

| 支柱 | 含义 | 类比 |
|------|------|------|
| **封装** | 隐藏实现，暴露接口 | 遥控器（按键可见，电路隐藏） |
| **继承** | 子类复用父类属性和方法 | 子类 extends 父类 |
| **多态** | 同一接口，不同实现 | `shape.area()` → Circle / Square |
| **抽象** | 提取共性，定义契约 | `abstract class Shape { area() }` |

## 🔢 FP 四大核心

| 概念 | 含义 | 示意 |
|------|------|------|
| **纯函数** | 相同输入 → 相同输出，无副作用 | `def add(a,b): return a+b` |
| **不可变性** | 数据创建后不修改 | 创建新列表而非 `list.append` |
| **高阶函数** | 函数可做参数/返回值 | `map`, `filter`, `reduce` |
| **函数组合** | 小函数组合成复杂流程 | `double(add_one(x))` |

---

## 🎯 范式选择指南

| 因素 | 推荐范式 |
|------|---------|
| **业务建模** | OOP（实体关系明确） |
| **数据转换** | 函数式（管道式处理） |
| **性能关键** | 命令式/过程式（可控） |
| **并发需求** | 函数式（无共享状态） |
| **大型项目** | OOP（易于扩展维护） |
| **快速原型** | 声明式/函数式（简洁） |

**实践建议**：多范式编程 — 用 OOP 组织系统骨架，用 FP 处理数据流，用声明式写查询。

## 💡 同一问题的不同视角

```
"从列表筛选偶数并求和"

命令式:  for→if→total+=  循环累加
函数式:  filter(偶)→sum  管道组合
OOP:     obj.sum_evens() 对象方法
声明式:  SQL: SELECT SUM ... WHERE even
```
