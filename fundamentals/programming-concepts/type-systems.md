# Type Systems - 类型系统

> 类型是什么？类型系统如何帮我们写出更好的代码？

## 🎯 核心概念

**类型** = 数据分类，定义了值范围、可用操作、内存表示。

**类型系统** = 语言用来检查和强制类型规则的机制。

---

## 📊 基本类型分类

| 分类 | 说明 | 示例 |
|------|------|------|
| **原始类型** | 语言内置基础类型 | int, float, bool, char |
| **复合类型** | 多个值组合 | 数组、列表、字典、元组 |
| **自定义类型** | 程序员定义 | 类、枚举、结构体 |
| **函数类型** | 函数的输入/输出类型 | `(int) → str` |

---

## 🔍 类型系统维度

### 维度一：静态 vs 动态

| 特性 | 静态类型 | 动态类型 |
|------|----------|----------|
| **检查时机** | 编译时 | 运行时 |
| **发现错误** | 编译期捕获 | 运行期才暴露 |
| **IDE 支持** | 强（自动完成/重构） | 弱 |
| **代码冗长度** | 高 | 低 |
| **灵活性** | 低 | 高 |
| **代表语言** | Java, C++, Rust, TS | Python, JS, Ruby |

### 维度二：强 vs 弱 (光谱)

```
强类型 ←──────────────────────→ 弱类型
Haskell  Python  Java  C  JavaScript
```

| 特性 | 强类型 | 弱类型 |
|------|--------|--------|
| **类型转换** | 严格，需显式 | 宽松，隐式 |
| **例子** | `"3" + 5 → TypeError` | `"3" + 5 → "35"` |

### 维度三：类型推导

编译器/解释器自动推断变量类型，兼顾安全与简洁：
```
let x = 42         → 自动推导为 int
let name = "Alice" → 自动推导为 str
```

## 🎭 泛型与多态

| 多态类型 | 说明 | 示例 |
|----------|------|------|
| **参数多态 (泛型)** | 类型参数化，编写通用代码 | `List[T]`, `Box<T>` |
| **子类型多态** | 子类替代父类 | `Animal`, `Dog extends Animal` |
| **Ad-hoc 多态** | 同名函数不同实现 | 函数重载 |

## 🛡️ 类型安全与高级类型

| 概念 | 说明 |
|------|------|
| **Option/Maybe** | 消灭空指针，`Some(value) \| None` |
| **Result** | 表达可失败操作，`Ok(v) \| Err(e)` |
| **联合类型** | 值可以是几种类型之一 | `int \| str` |
| **交叉类型** | 值必须满足多个类型 | `A & B` |
| **字面量类型** | 限定为精确值 | `"north" \| "south"` |
| **类型别名** | 复杂类型起个名字 | `Point = Tuple[float, float]` |

## 🎯 选择指南

| 需求 | 推荐 |
|------|------|
| **大型项目/团队协作** | 静态类型（早期发现错误，接口清晰） |
| **快速原型/脚本** | 动态类型（快速迭代，灵活性） |
| **性能关键** | 静态类型（编译优化） |
| **API 设计** | 静态类型（自带文档） |

**渐进式实践**：即使在动态语言中也使用类型注解 → 从无到有逐步精确。
