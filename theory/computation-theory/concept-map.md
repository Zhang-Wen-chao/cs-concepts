# 计算理论概念导航图 (Computation Theory)

> 探索计算的本质：什么能被计算，如何被计算，计算的边界在哪

## 🎯 核心思想

计算理论研究计算的数学基础，回答三个根本问题：
1. **什么能被计算？** — 可计算性理论
2. **如何描述计算？** — 自动机和形式语言
3. **计算的限制在哪？** — 停机问题等不可判定问题

---

## 🤖 自动机层次

| 自动机 | 记忆 | 对应语言 | 典型应用 |
|--------|------|---------|---------|
| **DFA/NFA** | 有限状态 | 正则语言 | 正则表达式、词法分析器 |
| **PDA**（下推自动机） | 栈(LIFO) | 上下文无关语言 | 括号匹配、语法分析 |
| **LBA**（线性有界自动机） | 有限纸带 | 上下文相关语言 | 更复杂的识别 |
| **图灵机** (TM) | 无限纸带 | 递归可枚举 | 通用计算模型 |

**能力递增：** DFA/NFA ⊂ PDA ⊂ LBA ⊂ 图灵机

### 核心定理
- **NFA ↔ DFA 等价**（子集构造法：k 状态 NFA → 最多 2ᵏ 状态 DFA）
- **PDA ↔ CFG 等价**（下推自动机与上下文无关文法等价）
- **Church-Turing 论题：** 图灵机 = 可计算极限，等价于 Lambda 演算、递归函数、现代编程语言

### 应用映射
```
正则表达式引擎: regex → NFA → DFA → 高效匹配
编译器词法分析:  Token 识别 → DFA
编译器语法分析:  CFG → LL(1)/LR(1) 解析器
协议状态机:     TCP 状态、游戏 AI、UI 导航
```

## 📜 乔姆斯基层次

| 类型 | 文法 | 自动机 | 示例语言 |
|------|------|--------|---------|
| **Type 3** | 正则文法 | DFA/NFA | (01)\*, `[0-9]{3}-[0-9]{4}` |
| **Type 2** | 上下文无关文法 | PDA | 0ⁿ1ⁿ, 括号匹配, 算术表达式 |
| **Type 1** | 上下文相关文法 | LBA | aⁿbⁿcⁿ |
| **Type 0** | 无限制文法 | 图灵机 | 任何可计算语言 |

**泵引理：** 用于证明某语言不属于某类型。典型：{0ⁿ1ⁿ} 非正则、{aⁿbⁿcⁿ} 非上下文无关。

## 🚫 可计算性边界

| 分类 | 存在算法？ | 行为 |
|------|-----------|------|
| **递归/可判定** | ✅ | 总能停机给出 yes/no |
| **递归可枚举/半可判定** | ✅ | yes 能停机，no 可能不停机 |
| **不可计算** | ❌ | 不存在任何算法 |

**经典不可判定问题：**
```
停机问题        — 程序会停机吗？
程序等价性      — 两个程序功能一样吗？
全程序正确性    — 程序有 bug 吗？
完美病毒检测    — 是病毒/恶意软件吗？
```

### 停机问题（图灵的证明核心）

假设存在 `HALT(P, I)`，构造 `TROUBLE(P)`: 若 HALT(P,P) 为真则死循环，否则停机。
问 `TROUBLE(TROUBLE)` 会停机吗？→ 矛盾！所以 HALT 不存在。

### Rice 定理

> 关于图灵机语言的**任何**非平凡性质都不可判定。

**实际后果：** 静态分析必然有误报/漏报；自动程序验证需要人工辅助；编译器优化有理论不完美性。

## 🔗 概念关系图

```
计算的三个层次
────────────────────────────────
自动机理论 ──→ DFA ⊂ PDA ⊂ 图灵机
    ↓
形式语言     ──→ 正则 ⊂ 上下文无关 ⊂ 上下文相关 ⊂ 递归可枚举
    ↓
可计算性     ──→ 可判定 > 半可判定 > 不可判定
    ↓
实际影响     ──→ 编译器(自动机)、静态分析(Rice)、验证(不可判定)
```
