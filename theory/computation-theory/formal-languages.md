# 形式语言 (Formal Languages)

> 用数学方式描述语法规则

## 🎯 核心思想

形式语言 = **字符串的集合** + **文法规则**来描述这些字符串的结构。

```
语言 L ⊆ Σ*（字母表 Σ 上所有有限字符串的子集）
文法 G = (V, Σ, R, S) — 从起始变量 S 用产生式规则 R 推导出字符串
```

---

## 📊 乔姆斯基层次 (Chomsky Hierarchy)

| 类型 | 文法限制 | 自动机 | 示例语言 | 封闭性（交/补） |
|------|---------|--------|---------|---------------|
| **Type 3** 正则 | `A → aB \| a` | DFA/NFA | `(01)*`、`a*b+` | ✅ ✅ |
| **Type 2** 上下文无关 | `A → α` | PDA | `0ⁿ1ⁿ`、括号匹配 | ❌ ❌ |
| **Type 1** 上下文相关 | `αAβ → αγβ` | LBA | `aⁿbⁿcⁿ` | ✅ ❓ |
| **Type 0** 递归可枚举 | `α → β`（无限制） | 图灵机 | 任何可计算 | ❌ ❌ |

### Type 3 — 正则语言

三种等价表达：**正则文法 = 正则表达式 = DFA/NFA**

```
正则表达式: (01)*
正则文法: S → 0A | ε, A → 1S
DFA:
       0          1
  → (S) ───→ (A) ───→ (S)
```

**泵引理（证明非正则）：** 存在泵长度 p，长串可分解为 `xyz`，`xyⁱz` 全在语言中。例：`0ⁿ1ⁿ` 不是正则的（泵 0 导致数量不匹配）。

### Type 2 — 上下文无关语言

**CFG + PDA 等价。** 关键性质：支持递归嵌套结构。

```
例：0ⁿ1ⁿ         S → 0S1 | ε
例：括号匹配     S → SS | (S) | ε
例：算术表达式   E → E+T | T,  T → T*F | F,  F → (E) | id
```

**二义性：** 同一个串有多个分析树。例：`E → E+E | E*E | id` 中 `id+id*id` 有两种解释。解决：引入优先级层次。

**泵引理（证明非 CFL）：** `aⁿbⁿcⁿ` 不是 CFL（泵后会打乱数量关系）。

### Type 1 — 上下文相关语言

`aⁿbⁿcⁿ` 的 CSG 示意：`S → aSBC | aBC`, `CB → BC`, `aB → ab`, ... 需要相邻符号交换规则。

### Type 0 — 递归可枚举语言

图灵机可识别的所有语言。包含所有可计算问题，但不包括某些不可判定问题的补集。

## 🔗 概念关系图

```
层次嵌套（能力递增）：
递归可枚举 ⊇ 上下文相关 ⊇ 上下文无关 ⊇ 正则
  (Type 0)     (Type 1)     (Type 2)    (Type 3)

应用映射：
正则语言    ←→ 词法分析（关键字、标识符、数字）
上下文无关  ←→ 语法分析（语句结构、AST）
上下文相关  ←→ 语义分析（类型检查、作用域）

CFL 封闭性：
✅ 并、连接、星号
❌ 交、补（不封闭！）

LL(k) / LR(k) 解析是 CFG 的子集，用于实际编译器
```
