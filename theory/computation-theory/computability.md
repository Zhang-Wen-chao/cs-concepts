# 可计算性理论 (Computability Theory)

> 探索计算的边界：什么能算，什么算不了

## 🎯 核心思想

**有些问题本质上是不可计算的**——再强大的计算机也解决不了。这不是技术限制，而是数学必然。

```
可计算 = 存在算法 = 可以写程序计算
不可计算 = 不存在算法 = 任何程序都会在某些输入上失败
```

---

## 📊 可判定性分类

| 类别 | 含义 | 示例 |
|------|------|------|
| **可判定**（递归语言） | 图灵机总停机，正确回答 yes/no | DFA 接受问题、正则语言空性、两个 DFA 等价 |
| **半可判定**（递归可枚举） | yes 可停机，no 可能不停机 | TM 接受问题、停机问题的 yes 实例 |
| **不可判定** | 不存在任何算法 | 停机问题、TM 等价性、CFG 二义性、Post 对应问题 |

### Church-Turing 论题

所有合理的计算模型（图灵机、Lambda 演算、递归函数、Python/C/Java）能力等价。如果某问题图灵机解决不了，任何计算设备都解决不了。

## 🚫 停机问题（核心不可判定问题）

**问题：** 给定程序 P 和输入 I，P 在 I 上会停机吗？

**图灵的证明（对角化论证）（简略）：**

```
假设存在 HALT(P, I) 能判断。
构造 TROUBLE(P): if HALT(P, P) → while True（对着干）
             else → return（配合）

问 TROUBLE(TROUBLE) 停机吗？
  若停机 → HALT 说停机 → TROUBLE 执行 while True → 不停机 → 矛盾
  若不停机 → HALT 说不→ TROUBLE 执行 return → 停机 → 矛盾
∴ HALT 不存在。
```

**直接推论（归约）：**

| 不可判定问题 | 归约来源 |
|-------------|---------|
| TM 接受问题 | 停机问题 ≤ TM 接受 |
| TM 语言空性 | 接受问题 ≤ 空性 |
| 两个 TM 等价 | 空性 ≤ 等价 |
| CFG 二义性 | PCP ≤ CFG 二义性 |
| 病毒检测 | 停机问题 ≤ 病毒检测 |

## 📜 Rice 定理

**关于图灵机语言的任何非平凡性质都是不可判定的。**

"非平凡" = 有些 TM 满足，有些不满足。

由此推出以下程序分析问题全不可判定：
- 程序 P 是否计算阶乘函数
- 程序 P 是否总停机
- 两个程序是否等价
- 死代码检测（某段代码是否永远不执行）
- 程序是否有安全漏洞
- 程序是否调用某个函数

## 🔗 概念关系图

```
所有语言
   ├── 递归可枚举（半可判定，TM 可识别）
   │     ├── 递归（可判定，总停机）
   │     │     └── P（多项式时间可解）
   │     └── 不可判定（如停机问题本身）
   └── 非递归可枚举（无法被任何 TM 识别）

归约链：停机问题 ≤ TM接受 ≤ TM空性 ≤ TM等价性 ≤ ...

实际影响：
  软件测试：只能证明 bug 存在，不能证明不存在
  静态分析：只能做近似分析，必有误报或漏报
  病毒检测：完美杀毒软件不存在
  程序验证：无法自动验证所有性质，需人工辅助（循环不变式）
  图灵完备：表达力越强，可判定性质越少
```
