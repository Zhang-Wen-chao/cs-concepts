# 自动机理论 (Automata Theory)

> 抽象的计算模型：用数学方式描述"计算"

## 🎯 核心思想

自动机是抽象的"状态机器"，根据输入在状态间跳转，判断输入是否合法。从最简单到最强大，能力逐级递增：

```
DFA/NFA  ⊂  PDA  ⊂  线性有界自动机  ⊂  图灵机
有限记忆    栈记忆    有限纸带            无限随机访问
```

---

## 📊 自动机层次

| 自动机 | 能力 | 对应语言 | 记忆 | 典型应用 |
|--------|------|---------|------|---------|
| DFA/NFA | 有限状态 | 正则语言 | 无（只有有限状态） | 正则表达式、词法分析 |
| PDA | 栈 | 上下文无关语言 | 栈（LIFO） | 括号匹配、语法分析 |
| 图灵机 | 通用计算 | 递归可枚举 | 无限纸带 | 任何算法 |

---

### DFA（确定性有限自动机）

定义：`(Q, Σ, δ, q₀, F)` — 对每个输入有唯一确定的下一个状态。

```
例：识别以"01"结尾的二进制串
       0           1
  → (q₀) ─────→ (q₁) ─────→ ((q₂))
      ↓           ↓
      └───────────┘      (其他情况回退)

运行：101 → q₀→q₁→q₂ → 接受 ✓
```

### NFA（非确定性有限自动机）

- 一个输入可有多个转移
- 支持 ε 转移（不读入字符跳转）
- 设计更直观，状态更少（但能力等价于 DFA）

**核心定理：NFA ↔ DFA 等价**（子集构造法：NFA 的 k 个状态 → DFA 最多 2ᵏ 个状态）

---

### PDA（下推自动机）

`DFA + 栈` → 能"计数"和"嵌套"。

```
例：识别 0ⁿ1ⁿ（相同数量的 0 和 1）
策略：读0压栈A，读1弹栈A，读完栈空 → 接受

运行：0011
栈: [] → [A] → [A,A] → [A] → [] → 接受 ✓

应用：括号匹配 ({[]})、算术表达式解析
```

**核心定理：PDA ↔ CFG 等价**

---

### 图灵机

`自动机 + 无限纸带 + 读写头` → 通用计算模型。

定义：`(Q, Σ, Γ, δ, q₀, q_accept, q_reject)`
- δ：`Q × Γ → Q × Γ × {L, R}`（读→写→移动）

```
例：识别 0ⁿ1ⁿ
纸带：[0][0][1][1][⊔]
策略：标记第一个0为X，找第一个1标记为Y，重复，直至全标记
```

**Church-Turing 论题：** 任何可计算的东西，图灵机都能计算。等价于 Lambda 演算、递归函数、现代编程语言。

---

## 🔗 概念关系图

```
自动机层次（能力递增）
─────────────────
DFA/NFA            → 正则语言       → 正则表达式 a*b+      → 词法分析
    │
PDA                → 上下文无关语言  → 括号匹配 0ⁿ1ⁿ      → 语法分析
    │
图灵机              → 递归可枚举语言 → 任何可计算问题      → 通用计算机
    │
    └── 限制：停机问题不可判定（详见 computability.md）

实际应用：
正则表达式引擎: regex → NFA → DFA → 高效匹配
编译器词法分析: Token 识别用 DFA
协议状态机: TCP 状态 (CLOSED→LISTEN→SYN_SENT→...)
```
