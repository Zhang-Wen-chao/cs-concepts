# 近似算法 (Approximation Algorithms)

> 当完美太贵时，追求"足够好"

## 🎯 核心思想

NP 完全问题无法快速求最优解 → **在多项式时间内找"接近最优"的解，有质量保证。**

**近似比 ρ**：最小化问题中 `近似解 ≤ ρ × 最优解`（最大化 `近似解 ≥ (1/ρ) × 最优解`）。ρ 越接近 1 越好。

```
完美（精确算法 ρ=1）  ↔  FPTAS（任意精度多项式） ↔  PTAS（精度指数时间）  ↔  常数近似  ↔  对数近似
```

---

## 📊 经典近似算法一览

| 问题 | 算法 | 近似比 | 核心思想 | 时间复杂度 |
|------|------|--------|----------|-----------|
| 顶点覆盖 | 贪心边匹配 | 2 | 选边加两端点，删关联边 | O(E) |
| 度量 TSP | MST+DFS 走捷径 | 2 | MST ≤ OPT，双倍边+三角不等式 | O(V²) |
| 度量 TSP（改进）| Christofides | 1.5 | MST + 奇度顶点最小完美匹配 | O(V³) |
| 集合覆盖 | 贪心 | H(n) ≈ ln n | 每步选覆盖最多新元素的子集 | O(U·S) |
| 0-1 背包 | FPTAS（缩放价值） | 1+ε | 价值缩放后 DP，时间 O(n³/ε) | O(n³/ε) |
| 装箱 (FFD) | 降序 First Fit | 11/9 ≈ 1.222 | 从大到小排序后 First Fit | O(n log n) |

### 不可近似性

- **一般 TSP**（无三角不等式）：没有常数近似比（除非 P=NP）
- **集合覆盖**：不存在 `(1-ε)ln(n)`-近似（除非 P=NP）

## 🧠 设计技巧速查

```
1. 贪心策略（顶点覆盖、集合覆盖、装箱）
2. 局部搜索（TSP 2-opt、3-opt）
3. 线性规划舍入（LP松弛 → 阈值舍入 → 2-近似）
4. 原始-对偶方法（顶点覆盖、Steiner树）
```

### 线性规划舍入示意（顶点覆盖）

```
ILP: min Σxᵥ  s.t. xᵤ+xᵥ≥1 ∀边(u,v), xᵥ∈{0,1}
LP松弛: xᵥ∈[0,1]  → 解出 LP 值
舍入: 若 xᵥ ≥ 0.5 则选 v
保证: 近似解 ≤ 2 × OPT
```

## 💡 实践建议

| 数据规模 | 推荐策略 |
|---------|---------|
| n < 30 | 精确算法（回溯/分支定界） |
| 30 < n < 10⁴ | 近似算法 / FPTAS |
| n > 10⁴ | 简单贪心 / 局部搜索 / 启发式 |
| 实时系统 | 最快的近似或启发式 |

**注意：** 理论近似比分析的是最坏情况，实际数据平均表现常常好得多（如 TSP 2-近似平均 ρ≈1.2-1.3）。

## 🔗 概念关系图

```
NP完全问题（精确解指数时间）
     │
     ├── 近似算法（有质量保证）── 近似比ρ
     │       │
     │       ├── PTAS：任意精度，时间O(n^(1/ε))
     │       └── FPTAS：任意精度，时间O(poly(n,1/ε))
     │
     ├── 启发式（无保证但实用）
     │       └── 模拟退火、遗传算法、局部搜索
     │
     └── 精确算法（小规模）
             └── 分支定界、回溯、DP
```
