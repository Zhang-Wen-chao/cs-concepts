# 时间与空间复杂度 (Time and Space Complexity)

> 衡量算法效率的标尺

## 🎯 核心思想

复杂度分析回答：**算法需要多少资源？**

| 维度 | 含义 |
|------|------|
| 时间复杂度 | 运行需要多少步 |
| 空间复杂度 | 运行需要多少内存 |

**为什么重要：** 查找 n=1000，线性查找 O(n) → 1000 步，二分查找 O(log n) → 10 步。同样的规模，效率差 100 倍。

---

## 📈 常见复杂度类

```
从快到慢：
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!)

性能估算（n=10⁶，每秒 10⁸ 次操作）：
O(1):        1步                  瞬间
O(log n):    20步                 瞬间
O(n):        10⁶步               0.01秒
O(n log n):  2×10⁷步             0.2秒
O(n²):       10¹²步              3小时
O(2ⁿ):       无法计算            宇宙年龄
```

| 复杂度 | 典型场景 | 示意 |
|--------|----------|------|
| O(1) | 数组访问、哈希表查 | `arr[i]` |
| O(log n) | 二分查找、平衡树 | 每次搜索空间减半 |
| O(n) | 遍历、线性查找 | `for x in arr` |
| O(n log n) | 归并/快速排序（平均） | 分治合并 |
| O(n²) | 冒泡/选择排序 | 嵌套循环 |
| O(2ⁿ) | 子集生成、朴素递归 Fib | 每个元素选/不选 |
| O(n!) | 全排列、TSP 暴力 | n! 种路径 |

## 🧠 大 O 记号

`f(n) = O(g(n))` 表示存在常数 c，当 n 足够大时 `f(n) ≤ c·g(n)`。忽略常数因子和低阶项。**关注的是增长趋势**，而非实际秒数。

### 分析技巧

```
单循环     → O(n)
嵌套循环   → 相乘 → O(n·m)
顺序语句   → 相加 → O(n+m) = O(max(n,m))
递归       → 主定理
```

### 主定理

对于 `T(n) = aT(n/b) + f(n)`：
- `f(n) = O(n^c)`, `c < log_b(a)` → `T(n) = O(n^(log_b(a)))`
- `f(n) = O(n^c log^k n)`, `c = log_b(a)` → `T(n) = O(n^c log^(k+1) n)`
- `f(n) = O(n^c)`, `c > log_b(a)` → `T(n) = O(f(n))`

例：归并排序 `T(n) = 2T(n/2) + O(n)` → `O(n log n)`

### 最好/最坏/平均

- **快速排序**：最好 O(n log n)，最坏 O(n²)（已排序+选首元素 pivot）
- **线性查找**：最好 O(1)，最坏 O(n)，平均 O(n)

## 💾 空间复杂度

| 级别 | 含义 | 示例 |
|------|------|------|
| O(1) | 几个变量 | 原地交换、迭代求和 |
| O(log n) | 递归栈深度 | 递归二分查找 |
| O(n) | 复制数组/哈希表 | 归并排序临时数组 |

### 时空权衡

斐波那契的演进：O(2ⁿ) 时间 O(n) 空间 → O(n) 时间 O(n) 空间（DP） → O(n) 时间 O(1) 空间（滚动变量）

**本质：用空间换时间，或用时间换空间。**

## 🔗 概念关系图

```
复杂度分析
   ├── 时间复杂度 ← 主定理、递归树、替换法
   └── 空间复杂度 ← 递归栈、辅助数据结构
         │
         └── 时空权衡 ← DP（空间换时间）、原地算法（时间换空间）

关联概念：
  数据结构选择：数组 O(1)访问 vs 链表 O(1)插入
  排序下界：比较排序 Ω(n log n)
  P vs NP：可计算但可能很慢
```
