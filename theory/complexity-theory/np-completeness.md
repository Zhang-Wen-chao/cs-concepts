# NP 完全性 (NP-Completeness)

> 困难问题的分类：P vs NP

## 🎯 核心思想

NP 完全性理论回答一个千禧年问题：**P = NP？**

```
P       = 能快速解决的问题（多项式时间）
NP      = 能快速验证答案的问题
NP完全  = NP 中最难的问题
NP-hard = 至少和 NP 中所有问题一样难
```

**震撼的事实：** 如果任何一个 NP 完全问题有多项式算法 → 所有 NP 问题都有多项式算法 → P = NP → 密码学崩溃。50 年了，没人找到，也没人证明不存在。

---

## 📊 复杂度类对比

| 类 | 定义 | 示例 | 算法 |
|----|------|------|------|
| P | 多项式时间可解 | 排序(O(n log n))、最短路径、素数判定(AKS) | 确定性多项式算法 |
| NP | 多项式时间可验证 | SAT、哈密尔顿回路、子集和、TSP | 验证器：多项式时间 |
| NP 完全 | NP ∩ NP-hard | SAT、3-SAT、顶点覆盖、TSP | 无多项式算法（已知） |
| NP-hard | 不比 NP 任何问题简单 | 停机问题（不可判定） | 可能比 NP 更难 |

### 经典 NP 完全问题

```
归约链（Cook-Levin 定理起步）：
SAT → 3-SAT → 独立集 ↔ 顶点覆盖 ↔ 团 → 哈密尔顿回路 → TSP → 子集和 → 背包
        ↓
     图着色（k≥3）

证明新问题是 NP 完全：
1. X ∈ NP（能多项式验证）
2. 已知 NP 完全问题 Y ≤ X（Y 归约到 X）
```

| 问题 | 输入 | 问题描述 |
|------|------|----------|
| SAT | 布尔公式 F | F 有满足赋值？ |
| 3-SAT | CNF，每子句3文字 | 可满足？ |
| 顶点覆盖 | 图 G，整数 k | 存在 ≤k 顶点覆盖所有边？ |
| 独立集 | 图 G，整数 k | 存在 ≥k 顶点互不相邻？ |
| 哈密尔顿回路 | 图 G | 存在经过所有顶点的回路？ |
| TSP | 城市+距离矩阵，目标 D | 存在 ≤D 的回路？ |
| 子集和 | 集合 S，目标和 t | 存在子集和为 t？ |
| 背包 | 物品(w,v)，容量 W，目标 V | 总重≤W 且总价≥V？ |
| 图着色 | 图 G，颜色数 k | 能用 k 色着色相邻不同？ |

## 🔧 如何应对 NP 完全问题

| 策略 | 适用场景 | 示例 |
|------|----------|------|
| 精确算法（回溯/分支定界） | n < 30 | 子集和回溯 O(2ⁿ) |
| 伪多项式 DP | 目标值小 | 子集和 DP O(n·target) |
| 近似算法（有质量保证） | 中等规模 | TSP 2-近似、顶点覆盖 2-近似 |
| 启发式（无保证但实用） | 大规模 | 模拟退火、贪心 TSP |
| 特殊情况多项式 | 受限输入 | 2-SAT (P)、树结构 |

### 近似算法快速参考

```
TSP 2-近似的本质：MST + DFS + 三角不等式走捷径
顶点覆盖 2-近似：任取一条边(u,v)，选两端，删关联边
```

## 🔗 概念关系图

```
可计算性（能不能算） → 复杂度（多快能算）→ P vs NP（千禧年问题）
                                                    │
                                                    ├── P = NP → 密码学崩溃
                                                    └── P ≠ NP → 永远需要近似/启发式

P ⊆ NP 显然成立（能快速解决必能快速验证）
P = NP?  未证明，多数计算机科学家认为 P ≠ NP
```
