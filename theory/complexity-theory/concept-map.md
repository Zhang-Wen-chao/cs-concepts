# 复杂度理论概念导航图 (Complexity Theory)

> 衡量问题的难度：多快能算，或者根本算不快

## 🎯 核心思想

复杂度理论研究算法效率和问题难度，回答三个问题：
1. **这个算法要多少资源？** — 时间与空间复杂度
2. **这个问题本质上多难？** — NP 完全性
3. **找不到完美算法怎么办？** — 近似算法

---

## ⏱ 时间与空间复杂度

| 记号 | 名称 | 典型算法 | n=10⁶ 时表现 |
|------|------|----------|-------------|
| O(1) | 常数 | 数组访问 | 瞬间 |
| O(log n) | 对数 | 二分查找 | 20 步 |
| O(n) | 线性 | 遍历 | 0.01s |
| O(n log n) | 线性对数 | 归并排序 | 0.2s |
| O(n²) | 平方 | 冒泡排序 | ~3 小时 |
| O(2ⁿ) | 指数 | 子集生成 | 宇宙年龄 |
| O(n!) | 阶乘 | 全排列 | 无法计算 |

**空间复杂度基准：** O(1)原地算法 → O(log n)递归栈 → O(n)哈希表/复制数组

**分析技巧：**
- 单循环 → O(n) | 嵌套循环 → 相乘 | 顺序语句 → 相加
- 递归：主定理 / 递归树 / 替换法
- 时空权衡：斐波那契 O(2ⁿ)→O(n)（用空间换时间）

### 数据结构复杂度速查

```
          访问   搜索   插入   删除
数组      O(1)   O(n)   O(n)   O(n)
链表      O(n)   O(n)   O(1)   O(1)
哈希表     N/A   O(1)   O(1)   O(1)
平衡树   O(log n) O(log n) O(log n) O(log n)
```

## 🧩 P vs NP — 问题难度分类

| 类 | 定义 | 示例 |
|----|------|------|
| **P** | 多项式时间可解 | 排序(O(n log n))、最短路径、素数判定、最大流 |
| **NP** | 多项式时间可验证 | SAT、哈密尔顿回路、TSP、子集和、背包 |
| **NP 完全** | NP ∩ NP-hard | SAT、3-SAT、顶点覆盖、TSP、图着色 |
| **NP-hard** | 不比 NP 任何问题简单 | TSP（判定版）、停机问题（不可判定） |

**核心归约链（Cook-Levin 起步）：**
```
SAT → 3-SAT → 独立集 ↔ 顶点覆盖 ↔ 团 → 哈密尔顿回路 → TSP → 子集和 → 背包
        ↓
     图着色 (k ≥ 3)
```

**千禧年问题 P vs NP：** 如果任一 NP 完全问题有多项式算法 → P = NP → 密码学崩溃。多数人认为 P ≠ NP。

---

## 🛠 应对 NP 完全问题的策略

| 策略 | 适用 | 示例 |
|------|------|------|
| 精确回溯/分支定界 | n < 30 | 子集和 O(2ⁿ) |
| 伪多项式 DP | 目标值小 | 子集和 O(n·target) |
| 近似算法（有保证） | 中等规模 | 顶点覆盖 2-近似、TSP 2-近似、Christofides 1.5-近似 |
| 启发式（无保证） | 大规模 | 模拟退火、Lin-Kernighan TSP |
| 特殊情况多项式 | 受限输入 | 2-SAT (P)、树结构 |

**近似算法设计工具箱：** 贪心策略、局部搜索(k-opt)、LP 舍入、原始-对偶

**FPTAS（完全多项式近似模式）：** 背包问题可在 O(n³/ε) 时间内得到 (1−ε) 最优解。

## 🔗 概念关系图

```
复杂度理论
│
├─ 资源度量 ── 时间复杂度(O(...))、空间复杂度、时空权衡
│
├─ 问题分类 ── P ⊆ NP ⊆ PSPACE，NP 完全为 NP 最难点
│
├─ 实用策略 ── ρ-近似 → PTAS → FPTAS → 启发式
│
└─ 算法竞赛速查
     n ≤ 10:  O(n!)     n ≤ 500:  O(n³)
     n ≤ 20:  O(2ⁿ)     n ≤ 10⁴:  O(n²)
     n ≤ 10⁶: O(n log n) 任意:  O(log n)/O(1)
```
