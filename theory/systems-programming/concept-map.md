# 系统编程核心概念图

> 范围：OS 概念 + 语言层概念，重点辨析"哪些是 C++ 独有的、哪些是通用的"。

## 一、四个核心概念在哪个"层"？

```
┌──────────────────────────────────────────────────────────┐
│  应用层 (Application Layer)                              │
│  - 你的 C++ / Python / Go / Java 代码                   │
├──────────────────────────────────────────────────────────┤
│  语言/标准库层 (Language/Stdlib Layer)                    │
│  - C++20 协程: co_await / co_yield / co_return          │
│  - C++ std::thread / std::atomic / std::mutex           │
│  - Python asyncio / Go goroutine / Rust tokio           │
│  - 智能指针、智能容器 (纯 C++ 概念，不依赖 OS)            │
├──────────────────────────────────────────────────────────┤
│  系统调用层 (System Call Layer)                          │
│  - pthread_create (线程)                                 │
│  - mmap / shm_open / shmat (共享内存)                    │
│  - pipe / socket / msg queue (IPC)                       │
│  - epoll (Linux) / kqueue (macOS/BSD) / IOCP (Windows)   │
├──────────────────────────────────────────────────────────┤
│  操作系统内核 (Kernel)                                  │
│  - 进程/线程调度、虚拟内存、文件系统、网络协议栈          │
└──────────────────────────────────────────────────────────┘
```

## 二、四个概念的真归属

| 概念 | 归属 | 谁定义 | 跨平台性 | 备注 |
|---|---|---|---|---|
| **协程 (Coroutine)** | **语言/库层** | 各语言自己 | 看语言 | C++20 才有，Python/Go 早就有了 |
| **共享内存 (Shared Memory)** | **OS 层** | POSIX | Unix-like 通用 | Linux/macOS/BSD 都有 |
| **进程间通信 (IPC)** | **OS 层** | POSIX | Unix-like 通用 | 6+ 种方式 |
| **epoll** | **OS 层（Linux 独有）** | Linux 内核 | **只 Linux** | macOS 用 kqueue，Windows 用 IOCP |

**关键判断**：
- 不是 C++ 独有 → 看具体概念
- 不是 Linux 独有 → 看具体概念
- **唯一有"语言绑定"的是协程**，C++20 之前 C++ 没有

## 三、C++ 在这四层里的位置

| 层 | C++ 的角色 |
|---|---|
| 应用层 | 写业务代码 |
| 语言/标准库层 | C++ 标准库实现，**C++ 的强项**（RAII、智能指针、模板、协程） |
| 系统调用层 | C++ **调用** OS API（`#include <sys/mman.h>` `mmap`） |
| OS 内核 | C/C++ 实现（Linux 内核就是 C 写的） |

**C++ 跨四层**：从最抽象的应用到最底层的系统调用，C++ 都能用。这也是 C++ 的"难"——**你要知道每层发生了什么**。

## 四、概念之间的关联

```
                    ┌─────────────────┐
                    │   协程 (用户态)  │  ← 语言层，单线程内多任务
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              ↓              ↓              ↓
        ┌──────────┐   ┌──────────┐   ┌──────────┐
        │  std::   │   │  std::   │   │  epoll/  │  ← 系统调用层
        │  thread  │   │  atomic  │   │  kqueue  │
        └────┬─────┘   └────┬─────┘   └────┬─────┘
             │              │              │
             ↓              ↓              ↓
        ┌──────────────────────────────────────┐
        │          操作系统内核                 │
        │  (进程/线程调度、内存管理、网络协议栈)  │
        └──────────────────────────────────────┘

        进程间通信 (IPC)  ← 横跨所有进程
        共享内存          ← 跨进程共享同一块物理内存
```

## 五、学习顺序建议

| 阶段 | 学什么 | 在哪一层 |
|---|---|---|
| Stage 0-1 | 编译、类型、值语义 | 语言/标准库层 |
| Stage 2-3 | RAII、虚函数表 | 语言/标准库层 |
| Stage 4-6 | 模板、STL、现代 C++ | 语言/标准库层 |
| **Stage 7** | **std::thread、std::atomic、内存序、协程** | **语言/标准库层** |
| Stage 7 末尾 | 进程、IPC、epoll/kqueue | 系统调用层 |
| 工作后/进阶 | 内核源码、驱动 | OS 内核 |

**原则**：**先语言层，再系统调用层**。原因：语言层概念（智能指针、模板）比系统调用（mmap、epoll）更可移植、更通用，先打好。

## 七、面试问题预设

| 问题 | 简答 |
|---|---|
| 协程是 OS 概念还是语言概念？ | **语言概念**，每种语言自己实现 |
| 共享内存是 C++ 提供的吗？ | 不是，**POSIX OS API** |
| epoll 是 C++ 的吗？ | 不是，**Linux 系统调用** |
| C++ 协程从哪个标准开始有？ | **C++20** |
| epoll 跨平台吗？ | **不跨平台**，macOS 用 kqueue，Windows 用 IOCP |
| 一个进程内的多个线程共享什么？ | **共享堆内存**（天然） |
| 不同进程怎么共享内存？ | **shm_open + mmap**（POSIX） |
