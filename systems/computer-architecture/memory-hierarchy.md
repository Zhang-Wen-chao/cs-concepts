# 存储层次结构 (Memory Hierarchy)

> 从寄存器到硬盘的完整存储体系

## 🎯 核心概念

**核心矛盾**：又快、又大、又便宜的存储不可兼得。解决方案：存储层次结构。

- **层次原则**：越靠近 CPU → 速度越快、容量越小、成本越高
- **局部性原理**：让层次结构有效的基础
- **Cache**：CPU 与内存之间的高速缓存，自动管理数据副本

### 层次对比

| 层级 | 容量 | 延迟 | 技术 |
|------|------|------|------|
| 寄存器 | <100B | <1ns | CPU 内部 |
| L1 Cache (I/D) | ~64KB | 1-2ns | SRAM |
| L2 Cache | ~512KB | 3-10ns | SRAM |
| L3 Cache (LLC) | ~8MB | 10-20ns | SRAM (多核共享) |
| 主内存 (DRAM) | ~16GB | 50-100ns | DRAM |
| SSD | ~1TB | 0.1ms | Flash |
| HDD | ~4TB | 5-10ms | 磁盘 |

---

## 🧩 局部性原理

| 类型 | 含义 | 示例 |
|------|------|------|
| 时间局部性 | 刚访问的数据很快再访问 | 循环变量、热点函数 |
| 空间局部性 | 访问某数据后邻域数据被接连访问 | 数组顺序遍历、结构体字段 |

**典型程序**：90% 的时间访问 10% 的数据。

## 💡 Cache 核心结构

### 地址分解（32位示例）

```
┌─────────────┬─────────────┬────────────┐
│  Tag        │  Index      │  Offset    │
│  标签位     │  索引位     │  偏移位    │
└─────────────┴─────────────┴────────────┘
  用于匹配      选择Cache组    块内定位
```

- **Cache Line**：基本存储单位，通常 64 字节
- **Hit**：数据在 Cache → 直接返回（快）
- **Miss**：数据不在 Cache → 从下一级加载整行（慢）

### 三种映射方式

| 方式 | 规则 | 硬件 | 命中率 | 应用 |
|------|------|------|--------|------|
| 直接映射 | 每个内存块只能到唯一位置 | 简单 | 低 | 小 Cache |
| 全相联 | 任意位置 | 复杂(并行比较器) | 高 | TLB |
| 组相联 (N-way) | Index 选组，组内随意 | 中等 | 中~高 | L1/L2/L3 主流 |

**实际**：L1 Cache 通常 8-way、L2 8~16-way、L3 16~20-way。

## 🔄 替换策略

| 策略 | 做法 | 特点 |
|------|------|------|
| LRU | 替换最久未使用的 | 命中率高，硬件开销大 |
| 伪LRU | 近似 LRU | L1/L2 常用 |
| FIFO | 替换最早进入的 | 实现简单，可能踢出热点 |
| Random | 随机替换 | 硬件极简，性能不稳定 |

---

## 📝 写策略

| 策略 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| 写直达 (Write-Through) | 同时写 Cache 和内存 | 一致性简单 | 写操作慢 |
| 写回 (Write-Back) | 只写 Cache，标记 Dirty | 写操作快 | 需 Dirty 位 |
| 写分配 (Write-Allocate) | Miss 时先加载到 Cache 再写 | Write-Back 适用 | — |
| 写不分配 (No-Write Allocate) | Miss 时直接写内存 | Write-Through 适用 | — |

---

## ⚡ TLB (Translation Lookaside Buffer)

**地址翻译的 Cache**——缓存最近使用的虚拟页→物理页映射。

- 容量小 (64~512 项)，全相联，速度极快 (1 cycle)
- **TLB Hit + Cache Hit**：~1-2 cycles（最快）
- **TLB Miss**：查页表 → 可能触发缺页异常（~200+ cycles）

```
虚拟地址 → TLB → 物理地址 → Cache → 数据
              ↓ Miss              ↓ Miss
            页表(内存)            主内存
```
