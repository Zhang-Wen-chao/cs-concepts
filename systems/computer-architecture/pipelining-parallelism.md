# 流水线与并行处理 (Pipelining and Parallelism)

> CPU 如何同时做多件事来提升性能

## 🎯 核心概念

**不让 CPU 任何部分闲置**——将指令执行拆分为多个阶段，重叠执行。

```
串行: I1:[IF][ID][EX][MEM][WB]      I2:        [IF][ID][EX][MEM][WB]
流水线: I1:[IF][ID][EX][MEM][WB]    I2:[IF][ID][EX][MEM][WB]    ...
```

- **理想吞吐**：每周期完成一条指令 (CPI=1)
- **理论最大加速比** = 流水线级数
- **实际受限**：冒险 (Hazards) 导致停顿

---

## 🏗️ 经典五级流水线

| 阶段 | 操作 |
|------|------|
| IF | 取指令 → IR |
| ID | 指令译码、读寄存器 |
| EX | ALU 运算、分支判断 |
| MEM | Load/Store 访存 |
| WB | 结果写回寄存器 |

---

## ⚠️ 流水线冒险

### 1. 结构冒险 — 硬件资源冲突
**解决**：哈佛架构 (I/D Cache 分离)、增加硬件、插入气泡

### 2. 数据冒险 — 指令间数据依赖

| 类型 | 含义 | 解决 |
|------|------|------|
| RAW (读后写) | ADD R1; SUB R1 需 R1 新值 | 数据转发 + 停顿 |
| WAR (写后读) | 反依赖，乱序执行中出现 | 寄存器重命名 |
| WAW (写后写) | 输出依赖，乱序执行中出现 | 寄存器重命名 |

**数据转发 (Forwarding)**：ALU/MEM 结果直接传给后续指令，无需写寄存器再读。
**Load-Use**：`LW R1; ADD R1, R2` → MEM 阶段才能转发，需停 1 周期。

### 3. 控制冒险 — 分支指令方向未知

| 策略 | 准确率 |
|------|--------|
| 预测不跳转 | ~60% |
| 1位预测器 | ~80-85% |
| 2位饱和计数器 | ~90-95% |
| 现代神经网络预测 | >97% |

---

## 🚀 超标量 & 乱序执行

| 模式 | 吞吐 | 特点 |
|------|------|------|
| 标量流水线 | 1 指令/cycle | 简单实现 |
| 超标量 (Superscalar) | N 指令/cycle | i7: 4-发射 |
| 乱序执行 (OoO) | 掩藏延迟 | Load 等待期间执行其他指令 |

**乱序执行关键技术**：
- **Tomasulo 算法**：保留站等待操作数就绪 + CDB 广播结果
- **重排序缓冲 (ROB)**：保证按序提交
- **寄存器重命名**：物理寄存器池 (100-200 个) 映射架构寄存器，消除 WAR/WAW

## 🧵 多核并行

- **~2005 前**：单核频率提升 → 功耗墙
- **2005 后**：多核架构，降低频率、并行提升吞吐

### Cache 一致性 — MESI 协议

| 状态 | 含义 | 条件 |
|------|------|------|
| M (Modified) | 仅本核有，已修改 | 独占可写 |
| E (Exclusive) | 仅本核有，未修改 | 独占只读 |
| S (Shared) | 多核共享，未修改 | 只读 |
| I (Invalid) | 无效 | 不可用 |

**总线监听**：所有核心监听总线，按 MESI 更新状态。

## 🔢 SIMD 并行

| 指令集 | 位宽 | 每周期操作 |
|--------|------|-----------|
| SSE | 128b | 4×float |
| AVX | 256b | 8×float |
| AVX-512 | 512b | 16×float |
| ARM NEON | 128b | 4×float |

**一条指令操作多个数据**：`c[0:3] = a[0:3] + b[0:3]` → 1 条 VADD。

---

## 🧊 超线程 (SMT)

复制 PC/寄存器文件模拟多个逻辑核心，共享 ALU/Cache。
- **性能增益**：~20-30%（利用流水线空闲周期），不是 2 倍

## 🔗 概念关系图

```
指令执行
  │
  串行 ──→ 流水线 (重叠阶段, CPI→1)
             │
        ├────┴────┐
      标量       超标量 (N/cycle + 乱序)
                         │
             ┌──────────┴──┐
          超线程(SMT)   多核 → MESI + Amdahl
```
