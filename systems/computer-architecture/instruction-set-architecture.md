# 指令集架构 (Instruction Set Architecture, ISA)

> 软件与硬件之间的契约

## 🎯 核心概念

**ISA 定义了软件能看到的硬件抽象**——是编译器目标与处理器设计的接口。

- **分离关注点**：ISA = 规范（做什么），微架构 = 实现（怎么做）
- **兼容性**：同一 ISA 的不同处理器可运行相同程序
- **ISA 定义**：指令格式、寄存器、寻址方式、数据类型

```
应用程序 (C/C++/Python)
       ↓ 编译
    汇编语言
       ↓ 汇编
┌─── ISA ◄── 这一层 ───┐
│  指令格式/寄存器/寻址  │
└────────────────────────┘
       ↓ 微架构实现
    硬件实现 (晶体管)
```

---

## ⚔️ CISC vs RISC

| 维度 | CISC (x86/x64) | RISC (ARM/RISC-V) |
|------|---------------|-------------------|
| 指令长度 | 不固定 (1-15B) | 固定 (通常 32b) |
| 译码 | 复杂 | 简单 |
| 流水线化 | 困难 | 容易 |
| 代码密度 | 高 | 低 |
| 能效比 | 低 | 高 |
| 编译器优化 | 较难 | 较易 |

**现代趋势**：x86 内部将 CISC 指令拆解为 RISC 风格的微操作(Micro-ops)再执行。

## 📐 指令格式

```
┌──────────┬──────────┬──────────┬──────────┐
│  操作码  │  目的    │  源操作  │  源操作  │
│  Opcode  │  Dest    │  数 1    │  数 2    │
└──────────┴──────────┴──────────┴──────────┘
```

### 指令类型
| 类型 | 用途 | 示例 (RISC-V) |
|------|------|--------------|
| 数据传送 | Load/Store | `LW x1, 0(x2)` |
| 算术逻辑 | 运算 | `ADD x1, x2, x3` |
| 控制转移 | 分支/跳转 | `BEQ x1, x2, label` |

---

## 🎯 寻址方式

| 方式 | 含义 | 典型场景 |
|------|------|---------|
| 立即数 | 数据在指令中 | 常量、小数值 |
| 寄存器 | 数据在寄存器 | 临时变量、循环变量 |
| 基址+偏移 | 寄存器+常量 | 数组、结构体 |
| 间接 | 指针指向 | 指针、动态数据 |
| PC 相对 | 相对当前指令 | 条件分支、PIC |

---

## 📋 寄存器组织

### RISC-V (32 个通用寄存器 + ABI 约定)

| 编号 | 名称 | 用途 |
|------|------|------|
| x0 | zero | 硬连线 0 |
| x1 | ra | 返回地址 |
| x2 | sp | 栈指针 |
| x5-7 / x28-31 | t0-6 | 临时 (调用者保存) |
| x10-17 | a0-7 | 参数/返回值 |
| x8-9 / x18-27 | s0-11 | 保存 (被调用者保存) |

### 调用约定
| 类型 | 寄存器 | 谁保存 |
|------|--------|--------|
| 调用者保存 | a0-7, t0-6 | 调用方压栈备份 |
| 被调用者保存 | s0-11 | 被调用方压栈备份 |

## 🔗 概念关系图

```
ISA (规范)
  │
  ├── CISC (x86) ──→ 复杂指令，硬件做更多
  │        │
  │        └── 译码 → 微操作 (RISC内核)
  │
  └── RISC (ARM/RISC-V) ──→ 简单指令，编译器组合
             │
         编译器重排/指令调度
             │
         流水线执行 ←→ 数据转发/分支预测
```
