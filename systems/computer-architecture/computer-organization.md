# 计算机组成原理 (Computer Organization)

> 冯·诺依曼架构与计算机的基本组成部件

## 🎯 核心概念

计算机组成研究计算机系统的**物理实现**——各部件如何协同工作执行程序。

- **存储程序**：指令和数据都存储在内存中，可被 CPU 读取执行
- **五大组件**：控制器、运算器(ALU)、存储器、输入设备、输出设备
- **核心循环**：取指(Fetch) → 译码(Decode) → 执行(Execute)

## 🏗️ 冯·诺依曼架构

```
        ┌─────────────────────────────────────┐
        │         控制器 (Control Unit)        │
        └──────────┬──────────────────────────┘
                   │ 控制信号
        ┌──────────▼──────────┐
        │   运算器 (ALU)       │◄──────┐
        │  算术运算/逻辑运算   │       │ 数据总线
        └─────────────────────┘       │
        ┌──────────────────────┐      │
        │   存储器 (Memory)     │◄─────┤
        │  指令+数据存储        │      │
        └──────────────────────┘      │
        ┌──────────────────────┐      │
        │   输入/输出设备        │──────┘
        └──────────────────────┘
```

## 🔧 CPU 核心部件

### 控制器 (CU)
| 组件 | 功能 |
|------|------|
| 程序计数器 (PC) | 存下一条指令地址，取指后自动递增 |
| 指令寄存器 (IR) | 存当前正在执行的指令 |
| 指令译码器 | 将指令翻译成控制信号 |

### 运算器 (ALU)
| 操作类型 | 示例 |
|---------|------|
| 算术运算 | 加、减、乘、除 |
| 逻辑运算 | AND、OR、NOT、XOR |
| 状态标志 | ZF(零)、CF(进位)、SF(符号)、OF(溢出) |

### 指令周期

```
┌──────────┐      ┌──────────┐      ┌──────────┐
│  取指    │ ───→ │  译码    │ ───→ │  执行    │
│  Fetch   │      │  Decode  │      │  Execute │
└──────────┘      └──────────┘      └──────────┘
     ▲                                     │
     └─────────────────────────────────────┘
```

- **取指**：PC → 内存 → IR，PC+1
- **译码**：分析操作码和操作数
- **执行**：运算/访存/跳转/I/O 之一

## 💾 存储器层次

| 层级 | 容量 | 延迟 | 特点 |
|------|------|------|------|
| 寄存器 | ~100B | <1ns | CPU 内部，最快 |
| L1 Cache | ~64KB | 1-2ns | SRAM，CPU 片上 |
| L2 Cache | ~512KB | 3-10ns | SRAM |
| L3 Cache | ~8MB | 10-20ns | 多核共享 |
| 主内存 | ~16GB | 50-100ns | DRAM |
| SSD | ~1TB | 0.1ms | 持久存储 |
| HDD | ~4TB | 5-10ms | 机械存储 |

### 字节序 (Endianness)
| 序型 | 示例 (0x12345678) | 平台 |
|------|-------------------|------|
| 大端序 | 12 34 56 78 (高位在低地址) | 网络字节序 |
| 小端序 | 78 56 34 12 (低位在低地址) | x86/x64 |

## 🔌 总线系统

| 总线类型 | 方向 | 功能 |
|---------|------|------|
| 数据总线 | 双向 | 传输数据，位宽=系统字长 |
| 地址总线 | 单向(CPU→外设) | 寻址，32位→4GB，64位→16EB |
| 控制总线 | 双向 | 读/写/中断/时钟信号 |

---

## ↔️ I/O 控制方式

| 方式 | 特点 | CPU 开销 |
|------|------|---------|
| 程序直接控制 (PIO) | CPU 轮询等待设备 | 高 |
| 中断驱动 (Interrupt) | 设备就绪后通知 CPU | 中 |
| DMA (直接内存访问) | DMA 控制器负责传输，CPU 仅发起和结束 | 低 |
