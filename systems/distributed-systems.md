# Distributed Systems - 分布式系统

> 多台机器协作，对外表现为一台机器

## 🎯 核心挑战

**分布式系统**是运行在多台机器上、通过网络通信、共享状态的程序集合。

```
        用户 1             用户 2
          │                  │
          ▼                  ▼
     ┌─────────┐       ┌─────────┐
     │ 节点 A   │──网络──│ 节点 B   │
     │  (有状态) │       │  (有状态) │
     └─────────┘       └─────────┘
          │
          ▼
        数据存储 (可能也是分布式的)
```

### 三大难题

| 问题 | 描述 | 原因 |
|------|------|------|
| **网络延迟** | 请求响应以毫秒计 | 物理距离 |
| **网络分区** | 节点间断开连接 | 交换机/光缆故障 |
| **节点故障** | 任何节点随时可能崩溃 | 硬件/软件/人为 |

**核心结果**：FLP 不可能定理 — 完全异步系统中，无法同时保证一致性、可用性、分区容忍性。

## 📚 核心概念

### 1. CAP 定理
最多同时满足两个：

```
一致性 (Consistency)    — 所有节点看到相同数据
可用性 (Availability)    — 每次请求必返回
分区容忍性 (Partition Tolerance) — 网络断开的系统仍能运行

        一致性
         /\
        /  \
       /    \
      / CAP  \
     /   / \   \
    /  CP    CA  \
   /              \
分区容忍性 ──────── 可用性
```

| 选择 | 含义 | 代表系统 |
|------|------|---------|
| **CP** | 分区时牺牲可用性，保证一致性 | Zookeeper、etcd |
| **AP** | 分区时牺牲一致性，保证可用性 | DynamoDB、Cassandra |
| **CA** | 不分区时（单机）兼具一致和可用 | 传统数据库 |

### 2. 一致性模型

| 模型 | 保证 | 性能代价 |
|------|------|---------|
| **强一致性** | 读到的总是最新的写 | 高（同步复制） |
| **最终一致性** | 不保证新鲜，但最终收敛 | 低（异步复制） |
| **因果一致性** | 有因果关系的操作有序 | 中 |
| **读己写** | 客户端总能读到自己的写 | 中 |

### 3. 共识算法 (Consensus)

让多个节点就一个值达成一致，即使部分节点故障。

| 算法 | 容错 | 特点 |
|------|------|------|
| **Paxos** | f ≤ n/2 | 理论上最优雅，但难以理解 |
| **Raft** | f ≤ n/2 | 易理解，Leader + 日志复制 |
| **PBFT** | f ≤ n/3 | 拜占庭容错（防恶意节点） |
| **PoW** | f < n/2 | 比特币，工作量证明 |

### Raft 三模块

```
选举 (Leader Election)  → 选出一个 Leader
日志复制 (Log Replication) → Leader 复制到 Follower
安全性 (Safety) → 保证只有拥有最新日志的节点才能当选
```

### 4. 分布式存储

| 系统 | 模型 | 一致性 | 应用 |
|------|------|--------|------|
| Zookeeper | CP | 强 | 配置管理、服务发现 |
| etcd | CP | 强 | Kubernetes 元数据 |
| Redis Cluster | AP | 最终 | 缓存 |
| Cassandra | AP | 最终（可调） | 时序数据 |
| DynamoDB | AP | 最终 | AWS 键值存储 |

## 💡 关键洞察

1. **故障是常态，不是异常** — 分布式系统假设任何组件都可能故障
2. **没有银弹** — 必须在 C/A/P 之间权衡，没有全都要的方案
3. **Raft > Paxos** — 实践中 Raft 因为易理解成为主流
4. **分布式 = 复杂的单机** — 所有单机问题（锁、事务、文件）在分布式下都更难
