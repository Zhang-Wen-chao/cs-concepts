# I/O Systems - I/O 系统

> 操作系统如何与外部设备通信？如何高效处理 I/O？

## 🎯 核心定义

**I/O 系统** — 操作系统中管理输入/输出设备的子系统，处理设备与内存之间的数据传输。

```
I/O系统 = 邮局系统
  设备驱动 = 邮局工作人员
  DMA = 自动分拣机
  缓冲区 = 邮箱
```

---

## 🖥️ I/O 设备分类

| 类型 | 特点 | 例子 |
|------|------|------|
| **块设备** | 以块为单位，可随机访问，有缓存 | HDD、SSD |
| **字符设备** | 以字节流，顺序访问，无缓存 | 键盘、鼠标、串口 |

---

## 🔌 I/O 控制方式

### 对比表

| 方式 | 原理 | ✅ | ❌ |
|------|------|----|----|
| **轮询** | CPU 不断检查设备状态 | 简单 | 浪费 CPU |
| **中断驱动** | 设备完成时发中断通知 CPU | 不占用等待 CPU | 频繁中断也开销大 |
| **DMA** | DMA 控制器直接传输数据，完成才中断 | CPU 完全解放 | 需专用硬件，竞争内存总线 |

```
轮询:     CPU 忙等设备就绪
中断:     CPU 做其他事 → 设备发信号 → 中断处理
DMA:      CPU 只说"传这 1MB"→ DMA 工作→ 一个中断通知完成
```

**DMA 最适合大量数据传输**（磁盘、网卡）。

---

## 🔧 设备驱动架构

```
应用程序 → 操作系统 VFS → 设备驱动 → 硬件
```

驱动提供统一接口：`open()` / `close()` / `read()` / `write()` / `ioctl()`，屏蔽硬件差异。

---

## 📦 I/O 缓冲

### 缓冲层次

| 策略 | 原理 | 场景 |
|------|------|------|
| **无缓冲** | 每次直接访问设备 | 简单场景 |
| **单缓冲** | 一个缓冲区暂存 | 基本的批量读写 |
| **双缓冲** | 两个轮流，一个填充一个传输 | 提高吞吐 |
| **循环缓冲** | 环形队列，生产者-消费者模式 | 流式数据 |

**为什么需要缓冲？** 设备速度差异大 → 减少系统调用 → 提高吞吐量。

## 💿 磁盘调度算法

### 物理结构

```
访问时间 = 寻道时间(最慢) + 旋转延迟 + 传输时间
```

### 算法对比

| 算法 | 思路 | ✅ | ❌ |
|------|------|----|----|
| **FCFS** | 按请求顺序 | 公平 | 寻道远路多 |
| **SSTF** | 选最近的请求 | 吞吐高 | 边缘磁道可能饥饿 |
| **SCAN (电梯)** | 来回扫描 | 无饥饿 | 中间磁道有利 |
| **C-SCAN** | 单向扫描+快速回零 | 更公平 | 开销略大 |

```
SCAN:  53 → 65 → 67 → 98 → 122 → 124 → 183 → (到顶) → 37 → 14
C-SCAN: 53 → 65 → ... → 183 → (回0) → 14 → 37
```
