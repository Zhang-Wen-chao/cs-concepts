# Processes and Threads - 进程与线程

> 操作系统中最核心的概念：程序如何运行？如何并发执行？

## 🎯 核心定义

**进程 (Process)** — 程序的一次执行实例，操作系统进行**资源分配和调度**的基本单位。**线程 (Thread)** — 进程内的一个执行流，**CPU调度**的基本单位。

### 生活类比

```
程序 (Program) = 菜谱  →  进程 (Process) = 做菜过程  →  线程 (Thread) = 一个厨师
```

---

## 📋 进程的组成与状态

### 进程组成

```
┌─────────────────────────┐
│   PCB (进程控制块)       │ ← 身份证：pid、状态、PC、寄存器
├─────────────────────────┤
│   代码段 / 数据段       │ ← 只读代码 + 全局变量
├─────────────────────────┤
│   堆 ↑  栈 ↓            │ ← 动态分配 & 函数调用
└─────────────────────────┘
```

### 🌊 进程状态转换

```
New → Ready ←→ Running → Terminated
              ↓ ↑ (I/O完成)
            Blocked
```

### 进程 vs 线程 对比表

| 特性 | 进程 | 线程 |
|------|------|------|
| **资源分配单位** | ✅ 独立地址空间 | ❌ 共享进程资源 |
| **CPU调度单位** | ❌ | ✅ |
| **创建/切换开销** | 大 | 小 |
| **通信方式** | IPC（管道、共享内存等） | 直接读写共享内存 |
| **崩溃影响** | 独立 | 拖垮整个进程 |

## 🚀 进程创建：fork() 模型

Unix 进程创建的"一招鲜"——**fork()**：调用一次，返回两次（父进程收到子进程 PID，子进程收到 0）。

```c
pid_t pid = fork();   // 复制当前进程
// 之后父子进程分道扬镳
```

结合 **exec()** 可在子进程中加载新程序，Python 的 `os.fork()` 同理。

## 🔀 进程调度算法

### 算法速览

| 算法 | 选谁？ | ✅ | ❌ |
|------|--------|----|----|
| **FCFS** | 先来先跑 | 简单 | 护航效应（短作业等长作业） |
| **SJF** | 选最短的 | 平均等待时间最优 | 长作业饥饿，需预知运行时间 |
| **RR** | 循环给时间片 | 响应好、公平 | 时间片选择困难，切换多 |
| **优先级** | 选优先级最高 | 灵活，适合实时 | 低优先级可能饥饿（可加老化） |

核心权衡：**响应时间 vs. 吞吐量 vs. 公平性**

## 🔄 上下文切换

进程/线程切换时：保存当前状态 → 加载目标状态。

```
进程切换：保存寄存器 + 切换页表 + 刷新 TLB  →  开销大
线程切换：只保存寄存器（共享地址空间）      →  开销小
```

---

## 🔗 进程间通信 (IPC)

| 方式 | 特点 |
|------|------|
| **管道** | 亲缘进程间单向数据流 (`pipe()`) |
| **共享内存** | 最快，需配合同步机制 |
| **消息队列** | 异步、解耦 |
| **信号** | 异步事件通知（软中断） |
| **套接字** | 跨网络通信 |

---

## 💡 选型原则

| 场景 | 推荐 |
|------|------|
| CPU 密集型、需隔离 | **多进程**（利用多核 + 安全性） |
| I/O 密集型、需频繁通信 | **多线程**（轻量、共享） |
