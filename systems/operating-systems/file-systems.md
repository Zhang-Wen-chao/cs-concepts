# File Systems - 文件系统

> 操作系统如何组织和管理磁盘上的数据？

## 🎯 核心定义

**文件系统** — 管理文件和目录的机制，负责文件的存储、检索、命名和保护。

```
文件系统 = 图书馆管理系统
  文件 = 书籍
  目录 = 书架/分类
  inode = 图书卡片（记录位置、大小等）
```

### 文件属性
**名称** · **类型** · **位置** · **大小** · **权限** · **时间戳** · **所有者**

## 📊 文件存储方式

| 方式 | 原理 | ✅ | ❌ |
|------|------|----|----|
| **连续分配** | 一个文件占连续磁盘块 | 顺序读快 | 外部碎片，难扩展 |
| **链接分配** | 块内指针链接到下一块 | 无外部碎片 | 随机访问慢，指针易损 |
| **索引分配** | 索引块存所有块指针 | 随机访问快，动态扩展 | 小文件也需索引块 |

**索引分配**最常用（如 ext4、NTFS）。

---

## 🔍 inode (索引节点)

Unix 文件元数据的核心结构：

```
inode #123
├── 大小 / 所有者 / 权限 / 时间戳
├── 12个直接指针 → 小文件直接读
├── 1个一级间接指针 → 指向索引块 → 128个数据块
├── 1个二级间接指针 → 128² 个数据块
└── 1个三级间接指针 → 128³ 个数据块
```

双重间接+三四重使 inode 能支持**非常大的文件**。

```bash
ls -i file.txt       # 查看 inode 号
stat file.txt        # 查看 inode 详细信息
```

## 💽 空闲空间管理

| 方式 | 原理 | 特点 |
|------|------|------|
| **位图 (Bitmap)** | 每位代表一个块，0=空闲 | 简单、易找连续块 |
| **空闲链表** | 空闲块链起来 | 空间效率高，但难找连续块 |

## 🔒 文件权限

### Unix 权限模型 (9位)

```
rwx rwx rwx
 │   │   └── 其他用户
 │   └────── 所属组
 └────────── 所有者
```

```bash
chmod 644 file.txt   # rw-r--r--
```

## 🗄️ 常见文件系统

| 系统 | 特点 |
|------|------|
| **FAT32** | 简单、兼容，用 FAT 表链式管理，无权限 |
| **ext4** | Linux 默认，inode + 日志 + 延迟分配 |
| **NTFS** | Windows 默认，MFT + ACL + 加密 + 压缩 |
| **ZFS** | Copy-on-Write + 快照 + 校验 + 自修复 |
