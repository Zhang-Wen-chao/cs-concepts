# Cryptography - 密码学

> 在不安全信道上的安全通信

## 🎯 核心目标

密码学的三个核心安全目标：

| 目标 | 含义 | 破解后果 |
|------|------|---------|
| **机密性 (Confidentiality)** | 只有授权方读懂内容 | 信息泄露 |
| **完整性 (Integrity)** | 内容未被篡改 | 恶意篡改 |
| **认证 (Authentication)** | 确认对方身份 | 身份冒充 |

## 📚 核心概念

### 1. 对称加密 (Symmetric)

```
          相同的密钥 K
      ┌──────────────────┐
      │                  │
      ▼                  ▼
明文 ──→ [AES加密] ──→ 密文 ──→ [AES解密] ──→ 明文
```

| 算法 | 密钥长度 | 特点 | 应用 |
|------|---------|------|------|
| **AES** | 128/256位 | 目前标准，快速安全 | TLS、磁盘加密 |
| **DES** | 56位 | 已淘汰（可暴力破解） | 只存在于遗产系统 |
| **ChaCha20** | 256位 | 软件实现快 | TLS、移动端 |

**问题**：密钥如何安全地传输给对方？

### 2. 非对称加密 (Asymmetric)

```
        公钥 Kpub          私钥 Kpriv
      ┌────────┐         ┌────────┐
      │        │         │        │
      ▼        │         ▼        │
明文 ──→ [RSA] ─┘     密文 ──→ [RSA] ──→ 明文
```

**关键特性**：公钥加密，私钥解密；私钥签名，公钥验签。

| 算法 | 安全长度 | 计算速度 | 应用 |
|------|---------|---------|------|
| **RSA** | 2048位 | 慢 | 密钥交换、数字签名 |
| **ECC** | 256位（≈RSA 3072） | 快 | HTTPS、区块链 |
| **Curve25519** | 256位 | 非常快 | 现代 TLS、Signal |

**核心价值**：解决了对称加密的密钥分发问题。A + B 用非对称交换对称密钥，之后用 AES 通信。

### 3. 哈希函数 (Hash)

```
输入 ──→ [SHA-256] ──→ 固定长度指纹 (256位)

特点：
  ✅ 不可逆：无法从哈希反推输入
  ✅ 抗碰撞：找不到两个不同输入有相同哈希
  ✅ 雪崩效应：改一个 bit，哈希完全不同
```

| 算法 | 输出长度 | 安全状态 | 应用 |
|------|---------|---------|------|
| **SHA-256** | 256bit | 安全 | TLS、比特币 |
| **SHA-3** | 可变 | 安全 | 新一代标准 |
| **MD5** | 128bit | 不安全（可碰撞） | 仅文件校验 |

**应用**：密码存储（加盐后哈希）、文件完整性校验、数字签名、工作量证明。

### 4. 数字签名 (Digital Signature)

```
签名：            私钥签名 ──→ 签名数据
验证：  数据 + 签名 ──→ 公钥验证 ──→ 通过/拒绝
```

**和手写签名等价**：签名人不可否认、签名不可伪造、数据不可篡改。

### 5. TLS 握手 (HTTPS 的基石)

```
Client                          Server
  │                                │
  │── TCP SYN ──────────────────→  │
  │←─ TCP SYN-ACK ──────────────── │
  │── TCP ACK ──────────────────→  │
  │                                │
  │── ClientHello ───────────────→ │  ← 支持的 TLS 版本、密码套件
  │←─ ServerHello + 证书 ──────────│  ← 服务器选好版本 + 发公钥证书
  │── 验证证书，生成 Pre-Master ──→│
  │←─ 确认，启用对称加密 ──────────│
  │                                │
  │────── 后续全用 AES 加密 ──────→│
```

**核心流程**：
1. 非对称加密验证身份（证书）
2. 非对称交换对称密钥（Pre-Master Secret）
3. 对称加密传输数据（AES）

### 6. 常见攻击

| 攻击 | 原理 | 防御 |
|------|------|------|
| **中间人攻击** | 冒充双方拦截通信 | 证书链验证 |
| **重放攻击** | 重新发送合法请求 | Nonce + 时间戳 |
| **生日攻击** | 利用碰撞概率找哈希冲突 | 增长哈希输出 |
| **侧信道攻击** | 通过时序/功耗推断密钥 | 常数时间实现 |
| **量子攻击** | 量子计算机破解 RSA/ECC | 后量子密码学 |

## 🔗 概念关系图

```
密码学
  │
  ├── 对称加密 (AES) ──→ 快速但密钥分发的难题
  │
  ├── 非对称加密 (RSA/ECC) ──→ 解决密钥分发，但慢
  │
  ├── 哈希 (SHA-256) ──→ 不可逆、抗碰撞
  │
  └── 组合 ──→ TLS 协议（非对称握手 + 对称传输）
                     │
                  HTTPS、SSH、VPN
```

## 💡 关键洞察

1. **"通过不安全信道建立安全信道"** — 这是密码学最反直觉但最有价值的地方
2. **AES + RSA = 现实** — 对称加密快用于数据，非对称加密慢用于密钥交换
3. **不要自己造密码** — 使用标准库（OpenSSL、libsodium）
4. **量子威胁** — RSA/ECC 在未来可能被量子计算机破解，后量子密码学正在标准化
