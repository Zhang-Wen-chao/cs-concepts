# Network Fundamentals - 网络基础与分层模型

> 分层降低复杂度，每层各司其职。

## 🎯 为什么需要分层

**网络通信太复杂**，需要分层来分解问题。

```
分层的好处：
✅ 每层专注自己的职责
✅ 层与层之间接口清晰
✅ 下层为上层提供服务
✅ 上层不关心下层细节
✅ 易于替换和升级
```

## 📊 OSI七层模型 vs TCP/IP四层模型

| OSI七层（理论） | 核心功能 | TCP/IP四层（实践） | 典型协议 |
|---|---|---|---|
| 7. 应用层 | 为应用程序提供网络服务 | 4. 应用层 | HTTP, DNS, FTP, SMTP |
| 6. 表示层 | 数据格式转换、加密 | ↑ | — |
| 5. 会话层 | 建立/管理/终止会话 | ↑ | — |
| 4. 传输层 | 端到端可靠传输 | 3. 传输层 | TCP, UDP |
| 3. 网络层 | 路由选择、寻址 | 2. 网络层 | IP, ICMP |
| 2. 数据链路层 | 帧传输、错误检测 | 1. 网络接口层 | Ethernet, Wi-Fi |
| 1. 物理层 | 物理信号传输 | ↑ | 双绞线、光纤 |

> OSI的会话层和表示层在实际中功能不明确，很多协议跨越多层，因此 TCP/IP 成为事实标准。实际工作中常用 **五层模型**（OSI 的上三层合并为应用层）。

## 📦 数据封装与解封装

```
层次            PDU名称          封装内容         添加的头部
────────────────────────────────────────────────────────────
应用层          Data             应用数据            —
传输层          Segment          应用数据        TCP/UDP 头
网络层          Packet           Segment            IP 头
数据链路层      Frame            Packet          帧头 + 帧尾
物理层          Bits             Frame               —

发送端：逐层向下封装（加头）
接收端：逐层向上解封装（去头）
```

### 封装过程示意

```
应用层: ┌──────────┐
        │ HTTP数据 │
        └────┬─────┘
              ↓ +TCP头
传输层: ┌──────┬──────────┐
        │TCP头 │ HTTP数据 │  ← Segment
        └──────┴────┬─────┘
                     ↓ +IP头
网络层: ┌──────┬──────┬──────────┐
        │IP头  │TCP头 │ HTTP数据 │  ← Packet
        └──────┴──────┴────┬─────┘
                            ↓ +帧头+帧尾
链路层: ┌─────┬──────┬──────┬──────────┬─────┐
        │帧头 │IP头  │TCP头 │ HTTP数据 │FCS  │  ← Frame
        └─────┴──────┴──────┴──────────┴─────┘
```

## 🔗 概念关系图

```
                  OSI七层模型（理论）
  ┌───────────────────────────────────────┐
  │ 应用层 (HTTP/DNS)                     │
  │ 表示层 (SSL/TLS)                      │
  │ 会话层 (RPC)                          │  ← 精简 →  TCP/IP四层模型
  │ 传输层 (TCP/UDP)  ←─────── 传输层 ───┤  ┌───────────────────┐
  │ 网络层 (IP)       ←─────── 网络层 ───┤  │ 应用层            │
  │ 数据链路层 (Ethernet, MAC)            │  │ 传输层 (TCP/UDP)  │
  │ 物理层 (双绞线, 光纤)                 │  │ 网络层 (IP)       │
  └───────────────────────────────────────┘  │ 网络接口层        │
                                             └───────────────────┘
                  ↓ 封装/解封装 ↓
      发送端: 应用层 → … → 物理层 → 比特流
      接收端: 比特流 → … → 应用层 → 数据
```

## 🔄 层间通信原则

- **同层通信**：使用相同协议，逻辑上直接通信，实际通过下层传递
- **相邻层通信**：通过服务访问点 (SAP) 和原语（Primitive）交互
