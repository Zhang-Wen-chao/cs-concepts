# 反向传播

## 一句话

从输出层的损失出发，**沿计算图倒着走**，每经过一个节点就把本地的导数乘上从上游传下来的梯度，最终算出每个参数对损失的贡献。

---

## 1. 要解决什么问题

正向传播很简单：输入 x，经过一系列计算，得到 ŷ，算出损失 L。

但问题是：**怎么知道每个参数 w 该调大还是调小、调多少？**

反向传播回答：**算 ∂L/∂w**——如果 ∂L/∂w > 0，说明 w 增大 → 损失增大，所以要减小 w。

---

## 2. 直觉：责任分配（Blame Assignment）

想象一条流水线，四个人依次加工一个产品，最后质检发现产品不合格：

```
A → B → C → D → 质检（损失）
```

要改进产品，你需要知道**每个环节的贡献**。

如果 D 做得不好，那责任主要在 D。但如果 D 没问题，那可能是 C 传给 D 的材料就有问题。

反向传播做的就是这件事：**从后往前，一层层看每层该负多大责。**

---

## 3. 链式法则

复合函数的求导：

```
L = f(g(h(x)))

∂L/∂x = ∂f/∂g · ∂g/∂h · ∂h/∂x
```

**外层导数 × 内层导数**，从外往里乘。

### 数值例子

```
x = 2
h(x) = 3x        → h = 6
g(h) = h + 1     → g = 7
f(g) = g²        → f = 49

∂f/∂g = 2g = 14     （外层：f 对 g 的导数）
∂g/∂h = 1           （中间：g 对 h 的导数）
∂h/∂x = 3           （内层：h 对 x 的导数）

∂f/∂x = 14 · 1 · 3 = 42
```

验证：f(x) = (3x+1)²，求导 = 2(3x+1)·3 = 6(3x+1)，x=2 时 = 42。✓

---

## 4. 一个完整的例子：线性回归

网络：`ŷ = w·x + b`，损失 MSE。

### 计算图

```
x → (× w) → z → (+) → ŷ → (MSE) → L
         b ↗
```

### 前向传播（具体数字）

```
x = 2, w = 1.5, b = 0.5, y = 5

z = w·x = 1.5 × 2 = 3
ŷ = z + b = 3 + 0.5 = 3.5
L = (ŷ - y)² = (3.5 - 5)² = 2.25
```

### 反向传播（算梯度）

从 L 开始，倒着走，每步算：**下游梯度 = 上游梯度 × 本地导数**。

```
第 1 步：∂L/∂ŷ = 2(ŷ - y) = 2(3.5 - 5) = -3
  → 解释：ŷ 增加 1，L 减少 3（因为 ŷ 比 y 小，增大 ŷ 会减小损失）

第 2 步：∂L/∂z = ∂L/∂ŷ × ∂ŷ/∂z = -3 × 1 = -3
  ∂L/∂b = ∂L/∂ŷ × ∂ŷ/∂b = -3 × 1 = -3

第 3 步：∂L/∂w = ∂L/∂z × ∂z/∂w = -3 × x = -3 × 2 = -6
```

### 更新参数

```
w_new = w - η × (-6) = 1.5 + 6η    ← 梯度负值，w 要增大
b_new = b - η × (-3) = 0.5 + 3η    ← 同样增大
```

检查一下逻辑：ŷ=3.5 < y=5，预测偏小了，所以需要增大 w 和 b 让 ŷ 变大。✓

## 5. 两层的完整例子

网络：`输入 x → 隐藏层(ReLU) → 输出层(sigmoid) → 损失 L`

### 前向传播

```
层 1（隐藏层）：
  z₁ = W₁·x + b₁        # 线性变换
  a₁ = ReLU(z₁)          # 激活

层 2（输出层）：
  z₂ = W₂·a₁ + b₂        # 线性变换
  ŷ = σ(z₂)              # sigmoid 激活

损失：
  L = -[y·log(ŷ) + (1-y)·log(1-ŷ)]   # 交叉熵
```

### 反向传播

从 L 开始，一步一步往回退。每层的计算模式都一样：

> **本层参数梯度 = 上游梯度 × 本层输入**

#### 输出层

```
∂L/∂ŷ = -(y/ŷ - (1-y)/(1-ŷ))

对于 sigmoid + 交叉熵，有一个简洁公式（建议直接记住）：
  ∂L/∂z₂ = ŷ - y

∂L/∂W₂ = ∂L/∂z₂ · a₁ᵀ    ← 上游梯度 × 本层输入
∂L/∂b₂ = ∂L/∂z₂           ← 偏置的梯度就是上游梯度
```

#### 从输出层传到隐藏层输出

```
∂L/∂a₁ = W₂ᵀ · ∂L/∂z₂    ← 误差按权重分配回每个隐藏神经元
```

这是关键一步：**输出层的误差 ∂L/∂z₂，乘 W₂ᵀ，映射回隐藏层的输出空间。**

#### 隐藏层

```
∂L/∂z₁ = ∂L/∂a₁ · ReLU'(z₁)
  → ReLU'(z) = 1 如果 z > 0，否则 0

∂L/∂W₁ = ∂L/∂z₁ · xᵀ
∂L/∂b₁ = ∂L/∂z₁
```

---

## 6. 核心模式

每一层的反向传播都遵循同一套模板：

```
               上游梯度
                  ↓
输入 → [线性层/激活层] → 输出
          ↑
      参数梯度 = 上游梯度 × 层输入

误差继续往下传 = 权重矩阵转置 × 上游梯度
```

| 层类型 | 前向 | 本地导数 | 传给上游的梯度 |
|---|---|---|---|
| Linear(W, b) | z = Wx + b | ∂z/∂W = x, ∂z/∂x = Wᵀ | Wᵀ · grad |
| ReLU | a = max(0, z) | 1 if z>0 else 0 | grad · (z>0) |
| Sigmoid | σ(z) | σ(z)(1-σ(z)) | grad · σ(z)(1-σ(z)) |
| 交叉熵 + Sigmoid | — | — | ŷ - y（简洁形式） |

---

## 7. 为什么高效：动态规划

如果神经网络有 28 层（像 Qwen3），naive 的做法是每层单独求导，大量重复计算。

反向传播的高效之处：**算完第 28 层的梯度后，第 27 层可以复用，第 26 层复用第 27 层的……**

```
L → 层28梯度 → 层27梯度 → 层26梯度 → ... → 层1梯度
    ← 一步步传，每步只算一次本地导数
```

总计算量 = 一次前向 + 一次反向，大约是前向的 2 倍。而不是层数 × 前向。

---

## 8. 如果面试官问

> **Q: 反向传播和梯度下降的区别？**
> A: 反向传播算梯度（∂L/∂w），梯度下降用梯度更新参数（w = w - η·∂L/∂w）。反向传播是"算"，梯度下降是"用"。

> **Q: 为什么用链式法则？**
> A: 因为网络是一层层叠起来的复合函数 L = f(g(h(x)))，要算 x 对 L 的影响，必须从外层一层层乘到内层。

> **Q: 为什么反向传播比前向求导高效？**
> A: 输出 L 是标量，参数有数百万。反向传播一次前向 + 一次反向算出所有参数的梯度，计算量约 2 倍前向。前向求导则需要每参数一次前向。
