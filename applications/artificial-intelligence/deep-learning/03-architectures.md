# 深度学习架构概览

## 一句话

三种主流架构分别擅长处理**网格**（CNN）、**序列**（RNN）、**集合**（Transformer）结构的数据。

---

## 对比

| 架构 | 擅长 | 核心操作 | 代表模型 |
|---|---|---|---|
| **CNN** | 图像、空间网格 | 卷积（局部连接 + 权值共享） | LeNet、ResNet、VGG |
| **RNN/LSTM** | 序列、时间线 | 隐状态循环（逐个 token 处理） | LSTM、GRU |
| **Transformer** | 序列、集合 | Self-Attention（看所有 token） | BERT、GPT、ViT |

---

## CNN

### 核心思路

图片的特征是局部的（边缘、纹理），不需要看完整张图。

```
输入图片 [3, 224, 224]
    ↓  卷积核 [3, 3, 3] 在图上滑动
特征图 [64, 222, 222]    ← 每个位置检测一种局部模式
    ↓  池化（降维）
特征图 [64, 111, 111]
    ↓  ...更多卷积层...
全连接 → 分类结果
```

### 关键特性

- **局部连接**：每个神经元只看输入的一小片区域
- **权值共享**：同一个卷积核在整个图上用同一套参数
- **平移不变性**：猫在图片左边还是右边，都能检测到

---

## RNN/LSTM

### 核心思路

序列数据有先后依赖关系，当前输出依赖于之前的信息。

```
RNN 单元：
hₜ = tanh(W·xₜ + U·hₜ₋₁ + b)

LSTM 单元：
添加"遗忘门"、"输入门"、"输出门"，控制信息保留和丢弃
```

### RNN 的问题

- 长序列中早期信息容易丢失（梯度消失）
- 必须串行处理，不能并行

### LSTM 的解决

门控机制让信息可以无损地沿时间轴传递，缓解长距离依赖问题。

---

## Transformer

### 核心思路

不再逐个 token 处理，而是**一次看所有 token**，用 Attention 决定每个 token 应该关注谁。

```
输入 token: [我, 爱, 猫]
    ↓
Self-Attention:
"爱" 需要关注 "我" 和 "猫" 各多少？
    ↓
加权融合 → 输出每个 token 的新表示
```

### Transformer vs RNN

| | RNN | Transformer |
|---|---|---|
| 处理方式 | 串行，逐个 token | 并行，一次全部 |
| 长距离依赖 | 容易丢失 | 任意距离 O(1) |
| 计算效率 | 慢（不能并行） | 快（可并行） |
| 位置信息 | 天然有序 | 需要额外加位置编码 |

---

## 为什么 Transformer 赢了

1. **可并行** → 训练快，能大规模
2. **长距离** → Attention 让任意两个 token 直接连接
3. **规模法则** → 参数越大、数据越多，效果越好（RNN/CNN 有瓶颈）

GPT、BERT、LLaMA、Qwen 都是 Transformer。
