# 深度学习架构概览

## 一句话

三种主流架构分别擅长处理**网格**（CNN）、**序列**（RNN）、**集合**（Transformer）结构的数据。

---

## 对比

| 架构 | 擅长 | 核心操作 | 代表模型 |
|---|---|---|---|
| **CNN** | 图像、空间网格 | 卷积（局部连接 + 权值共享） | LeNet、ResNet、VGG |
| **RNN/LSTM** | 序列、时间线 | 隐状态循环（逐个 token 处理） | LSTM、GRU |
| **Transformer** | 序列、集合 | Self-Attention（看所有 token） | BERT、GPT、ViT |

---

## CNN

### 权重在哪里

```
输入图片 [3, 224, 224]
    ↓
每个位置的 3×3 窗口 → [×卷积核] → 一个数字     ← 卷积核 = 权重
                       ↑
              形状 [3, 3, 3]，也叫 filter
              随机初始化，反向传播更新
    ↓
64 个不同卷积核，每个核学一种模式
    ↓
特征图 [64, 222, 222]
    ↓  池化降维
特征图 [64, 111, 111]
    ↓  ...更多卷积层...
全连接 → 分类结果
```

卷积核跟线性层的 W 是同一类东西——**可训练的参数**。区别：

```
线性层：每个输出连所有输入（"全"连接）
        矩阵形状 [输出维度, 输入维度]
        
卷积核：每个输出只看输入的 3×3 区域（"局部"连接）
        矩阵形状 [输出通道, 输入通道, 3, 3]
```

**训练时：反向传播算卷积核里每个数字的梯度 → 更新**
**64 个核初始随机 → 训练分化 → 有的学边缘、有的学纹理、有的学颜色**

### 关键特性

- **局部连接**：每个神经元只看输入的一小片区域
- **权值共享**：同一个卷积核在整个图上用同一套参数
- **平移不变性**：猫在图片左边还是右边，都能检测到

## RNN/LSTM

### 权重在哪里

```
处理第 t 个 token xₜ 时：

xₜ ──→ [× W] ──┐                          ← W: 权重 [hidden, input_dim]
                 ↓                          ← U: 权重 [hidden, hidden]
hₜ₋₁ ─→ [× U] ─→ + ──→ tanh ──→ hₜ       ← b: 偏置 [hidden]
                 ↑                          ← 都是随机初始化，反向传播更新
                 b
```

**hₜ = tanh(W·xₜ + U·hₜ₋₁ + b)**

RNN 的参数有两组：W（处理当前输入）和 U（处理上一次的记忆）。比 Attention 只多了个**递归**——当前时间步的输出 hₜ 同时作为下一个时间步的输入。

### LSTM 的权重在哪里

LSTM 在 RNN 基础上加了三个"门"，每个门都是一组独立权重：

```
遗忘门：fₜ = σ(Wf·xₜ + Uf·hₜ₋₁ + bf)     → 决定丢弃多少旧记忆
输入门：iₜ = σ(Wi·xₜ + Ui·hₜ₋₁ + bi)     → 决定写入多少新信息
输出门：oₜ = σ(Wo·xₜ + Uo·hₜ₋₁ + bo)     → 决定输出多少信息
候选：  c̃ₜ = tanh(Wc·xₜ + Uc·hₜ₋₁ + bc)
```

**LSTM 的参数 = RNN 的 4 倍**（4 组 W + 4 组 U），全是可训练参数，同样随机初始化→反向传播更新。门控让梯度有一条"直达通道"，缓解了梯度消失。

### RNN 的问题

- 长序列中早期信息容易丢失（梯度消失）
- 必须串行处理，不能并行

### LSTM 的解决

门控机制让信息可以无损地沿时间轴传递，缓解长距离依赖问题。

## Transformer

### 核心思路

不再逐个 token 处理，而是**一次看所有 token**，用 Attention 决定每个 token 应该关注谁。

```
输入 token: [我, 爱, 猫]
    ↓
Self-Attention:
"爱" 需要关注 "我" 和 "猫" 各多少？
    ↓
加权融合 → 输出每个 token 的新表示
```

### Transformer vs RNN

| | RNN | Transformer |
|---|---|---|
| 处理方式 | 串行，逐个 token | 并行，一次全部 |
| 长距离依赖 | 容易丢失 | 任意距离 O(1) |
| 计算效率 | 慢（不能并行） | 快（可并行） |
| 位置信息 | 天然有序 | 需要额外加位置编码 |

---

## 为什么 Transformer 赢了

1. **可并行** → 训练快，能大规模
2. **长距离** → Attention 让任意两个 token 直接连接
3. **规模法则** → 参数越大、数据越多，效果越好（RNN/CNN 有瓶颈）

GPT、BERT、LLaMA、Qwen 都是 Transformer。
