# Stage 1: C++ vs Python 本质差异

> 这一节是"小白到 C++ 程序员"真正开始写代码的第一节。  
> 重点：**C++ 跟 Python 最大的 5 个不同**。每个都讲机制 + 跑代码验证。

## 5 个本质差异

| 差异 | Python | C++ |
|---|---|---|
| 1. 类型系统 | 动态类型（运行时才知道） | **静态类型**（编译时就知道） |
| 2. 执行方式 | 解释执行（边解释边跑） | **编译执行**（先编译成机器码再跑） |
| 3. 内存管理 | 自动 GC（垃圾回收） | **手动 + RAII**（你管生命周期） |
| 4. 值传递 | 引用语义为主 | **值语义**（拷贝是默认行为） |
| 5. 错误处理 | 异常 + duck typing | **类型 + 异常 + 编译期检查** |

---

## 1. 静态类型：变量在声明时类型就定死了

**Python**（动态类型）：
```python
x = 42          # x 现在是 int
x = "hello"     # x 现在变成 str，OK
x = [1, 2, 3]   # x 又变 list，也 OK
```

**C++**（静态类型）：
```cpp
int x = 42;       // x 是 int，从今往后只能是 int
x = "hello";      // ❌ 编译错误：不能把 const char* 赋给 int
x = 3.14;         // ❌ 隐式转换也不行：int 不能存 3.14
```

**为什么这么设计**：C++ 把"类型检查"从**运行时**搬到**编译时**，bug 提前到写代码时就被抓住，**不需要等程序跑起来才发现**。

**面试常考**：
- "为什么 C++ 要静态类型？" → 性能（不需要运行时检查类型）+ 提前发现 bug
- "Python 不是也能加 type hint 吗？" → 是的，但**只是提示**，运行时**不会强制**

---

## 2. 编译执行 vs 解释执行

你 Stage 0 已经看过了。**最本质的差别**：

- **Python**：`python hello.py` → 启动 Python 解释器 → 一行一行翻译成字节码 → 跑
- **C++**：`clang++ hello.cpp -o hello` → 产生 hello（机器码）→ `./hello` → **直接被 CPU 执行**

**速度差异**：C++ 一般比 Python 快 10-100 倍，因为：
- 没有解释器这层开销
- 类型固定，CPU 能做很多优化
- 内存布局可控，cache 友好

---

## 3. 内存管理：谁负责释放

**Python**：
```python
def foo():
    data = [1, 2, 3]  # Python 解释器自动分配
    return data       # 引用计数 + GC 自动回收（你不用管）
```

**C++**：
```cpp
void foo() {
    int* p = new int[1000];  // 手动分配堆内存
    // ... 用 p ...
    delete[] p;              // 手动释放！忘了 = 内存泄漏
    // 异常抛出 → 跳过 delete → 泄漏
}
```

**核心问题**：谁负责释放？

- **Python**：GC（garbage collector）**自动**回收 → 你不用管
- **C++**：默认**没人管** → 你得自己处理，否则就泄漏

**C++ 的解法**：**RAII**（资源获取即初始化，后面 Stage 2 详讲），核心思想：**把资源的生命周期绑到对象的析构函数上**，离开作用域自动释放。

```cpp
// RAII 写法：把内存管到 std::vector 里
void foo() {
    std::vector<int> v(1000);  // 构造时分配
    // ... 用 v ...
}  // 离开作用域，自动调用 v 的析构函数，释放内存
// 哪怕中间抛异常，析构函数也保证被调用
```

**面试必问**：
- "什么是内存泄漏？怎么避免？" → 答：new 完忘了 delete。避免：智能指针 / RAII / 不直接用 new
- "Python 有内存泄漏吗？" → 答：有，但更难触发，主要是循环引用

## 4. 值语义：默认拷贝

**Python**（引用语义）：
```python
a = [1, 2, 3]
b = a         # b 和 a 指向同一个列表
b.append(4)
print(a)      # [1, 2, 3, 4] ← a 也变了！
```

**C++**（值语义）：
```cpp
std::vector<int> a = {1, 2, 3};
std::vector<int> b = a;  // b 是 a 的**副本**
b.push_back(4);
print(a);  // {1, 2, 3}      ← a 没变
print(b);  // {1, 2, 3, 4}   ← 只有 b 变了
```

**为什么会这样**：C++ 的 `=` 默认是**拷贝赋值**（除非显式用 `&` 或 `std::move`）。

**这带来的开销**：
```cpp
std::vector<int> big(1000000, 0);
std::vector<int> copy = big;  // 复制 100 万元素的数组！慢！
```

**C++ 的解法**：移动语义（`std::move`，Stage 2 详讲），**转移所有权而不是拷贝数据**。

**面试必问**：
- "C++ 拷贝构造和移动构造有什么区别？" → 答：拷贝是**深拷贝**（复制数据），移动是**所有权转移**（指针 swap，O(1)）
- "什么时候用拷贝什么时候用 move？" → 答：能 move 就 move（看后面 Stage 2）

## 5. 错误处理

**Python**（异常 + duck typing）：
```python
def add(a, b):
    return a + b
add(1, 2)        # OK
add("hello", 2)  # 运行时异常：TypeError
```

**C++**（编译期 + 异常）：
```cpp
int add(int a, int b) {
    return a + b;
}
add(1, 2);       // OK
add("hello", 2); // ❌ 编译错误：const char[6] 不能转 int
```

**关键差别**：C++ 的很多错误**编译时就抓到**，Python 只能**运行时**抓。

**C++ 的异常机制**：
```cpp
try {
    throw std::runtime_error("出事了");
} catch (const std::exception& e) {
    std::cerr << e.what() << "\n";
}
```

跟 Python 很像，但**C++ 不强制你捕获**（默认会调用 `std::terminate` 终止程序）。
